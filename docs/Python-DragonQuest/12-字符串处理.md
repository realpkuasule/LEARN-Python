## 核心讲解

### 12.1 字符串格式化——旁白字幕生成器

f-string 是 Python 最常用的字符串格式化方式。在 RPG 里，它就是你战斗画面的字幕引擎——固定句式 + 动态数据 = 即时旁白。

```python
# 战斗日志
hero_name = "勇者刘老三"
monster_name = "恶龙"
damage = 145
is_critical = True

log = f"{hero_name} 的传说之剑发出耀眼光芒！对 {monster_name} 造成了 {damage} 点伤害！"
if is_critical:
    log += " 暴击！！"
```

**逐行解读：**

- `hero_name = "勇者刘老三"` 和 `monster_name = "恶龙"` —— 定义角色名和怪物名，这些是可变的数据。
- `damage = 145` —— 伤害值是计算出来的结果。
- `is_critical = True` —— 布尔值标记是否暴击。
- `f"...{hero_name}...{monster_name}...{damage}..."` —— f-string 以 `f"` 开头，花括号 `{}` 内放入变量名，运行时自动替换为变量的值。
- `if is_critical: log += " 暴击！！"` —— 根据暴击标志，在字幕末尾追加暴击提示。`+=` 是字符串拼接的简写。

**类比**：战斗画面的旁白字幕——每个技能有一个描述模板，伤害数字和角色名填进去。f-string 就是字幕生成器。模板是死的，数据是活的，结合起来就活了。

**f-string 常用技巧**：

```python
# 数字格式化
print(f"暴击率：{0.15:.0%}")        # → 暴击率：15%
print(f"金币：{golds:,}")            # → 金币：3,840

# 表达式放在花括号里
print(f"剩余 HP：{hero['max_hp'] - hero['hp']} 点可回复")

# 对齐——战斗信息排版
print(f"{'勇者刘老三':>8} HP: {hero['hp']:>4}")
print(f"{'恶龙':>8} HP: {dragon['hp']:>4}")
```

### 12.2 字符串常用方法（上）——名字清洗与宝箱解析

玩家输入名字、宝箱掉落列表、装备名拼接——这些日常操作都离不开字符串方法。

```python
# 玩家输入名字时多打了空格
name = "  勇者刘老三  "
clean_name = name.strip()   # → "勇者刘老三"

# 格式化装备名
weapon = "勇者之剑"
rank = "+10"
full_name = weapon + "·" + rank   # → "勇者之剑·+10"

# 解析宝箱内容「回复药×3、魔力药×2、金币×500」
chest = "回复药×3、魔力药×2、金币×500"
items = chest.split("、")   # → ["回复药×3", "魔力药×2", "金币×500"]
```

**逐行解读：**

- `name = "  勇者刘老三  "` —— 模拟玩家起名时不小心在前后打了空格。
- `clean_name = name.strip()` —— `strip()` 去除字符串首尾的空白字符（空格、换行、制表符）。返回干净的名字。
- `weapon = "勇者之剑"` 和 `rank = "+10"` —— 装备基础名和强化等级。
- `full_name = weapon + "·" + rank` —— 用 `+` 拼接字符串，中间加上分隔符「·」。结果是 `"勇者之剑·+10"`。
- `chest = "回复药×3、魔力药×2、金币×500"` —— 宝箱内容是一串文字，用顿号分隔。
- `items = chest.split("、")` —— `split()` 按指定分隔符切割字符串，返回列表。这里按顿号切，得到 3 个物品字符串。

**类比**：玩家起名字时不小心打了空格——`strip()` 帮你去掉。宝箱掉落列表是一串文字——`split` 帮你切开成一个一个物品。

### 12.3 字符串常用方法（下）——技能书鉴定

在 RPG 世界里，鉴定魔法书、检查任务物品名——都需要字符串的查找和判断方法。

```python
# 技能书鉴定
skill_book = "禁咒·陨石术（远古文字）"

skill_book.startswith("禁咒")   # → True，这是禁术！
skill_book.find("陨石")         # → 3，在第 3 个位置
"（远古文字）" in skill_book    # → True，需要翻译
```

**逐行解读：**

- `skill_book = "禁咒·陨石术（远古文字）"` —— 一本技能书的全名。
- `skill_book.startswith("禁咒")` —— `startswith()` 检查字符串是否以指定文字开头。返回 `True`，说明这是禁术，需要特殊处理。
- `skill_book.find("陨石")` —— `find()` 返回子字符串第一次出现的位置索引。`"陨石"` 出现在索引 3 的位置（`"禁"=0, "咒"=1, "·"=2, "陨"=3`）。如果找不到则返回 `-1`。
- `"（远古文字）" in skill_book` —— `in` 运算符检查子字符串是否在字符串中。返回 `True`，说明这本书需要翻译才能阅读。

**类比**：鉴定魔法书——看名字就知道是不是禁术（`startswith`），需要不需要翻译（`find` 找「远古文字」标记）。

**常用字符串方法速查**：

| 方法                          | 作用             | 示例                                      |
| ----------------------------- | ---------------- | ----------------------------------------- |
| `str.strip()`                 | 去首尾空白       | `"  abc  ".strip()` → `"abc"`             |
| `str.split(sep)`              | 按分隔符切割     | `"a、b、c".split("、")` → `["a","b","c"]` |
| `str.startswith(x)`           | 是否以 x 开头    | `"禁咒·陨石".startswith("禁咒")` → True   |
| `str.endswith(x)`             | 是否以 x 结尾    | `"铁剑".endswith("剑")` → True            |
| `str.find(x)`                 | 查找 x 的位置    | `"abc".find("b")` → 1                     |
| `str.replace(a,b)`            | 替换所有 a 为 b  | `"史莱姆".replace("莱","拉")` → "史拉姆"  |
| `str.upper()` / `str.lower()` | 大小写转换       | `"Hero".upper()` → `"HERO"`               |
| `in` 运算符                   | 检查子串是否存在 | `"剑" in "铁剑"` → True                   |

### 12.4 正则表达式（上）——从战斗日志中提取数据

当字符串处理的需求超出简单方法的能力范围，正则表达式就该登场了。正则用模式匹配的方式在文本中查找、提取、替换内容。

```python
import re

# 从战斗记录里提取所有伤害数字
battle_log = "对史莱姆造成25点伤害！暴击造成50点伤害！普通攻击15点伤害"
damages = re.findall(r"造成(\d+)点伤害", battle_log)
# → ["25", "50", "15"]

# 检查密码强度（至少8位、含大小写和数字）
password = "Hero2024!"
if re.match(r"^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$", password):
    print("密码够强！可以保护你的存档了")
```

**逐行解读：**

- `import re` —— 导入 Python 的正则表达式模块。使用任何正则功能前必须导入。
- `battle_log = "对史莱姆造成25点伤害！暴击造成50点伤害！普通攻击15点伤害"` —— 一段典型的战斗记录文本。
- `re.findall(r"造成(\d+)点伤害", battle_log)` —— `findall()` 查找所有匹配的文本并返回列表。`r"..."` 是原始字符串，避免反斜杠被转义。
  - `造成` —— 匹配文字「造成」。
  - `(\d+)` —— `\d` 匹配一个数字，`+` 表示一个或多个，`()` 表示提取这一部分。所以提取的是「造成」和「点伤害」之间的数字。
  - `点伤害` —— 匹配文字「点伤害」。
- `re.match(r"^(?=.*[a-z])(?=.*[A-Z])(?=.*\d).{8,}$", password)` —— 密码强度检查正则：
  - `^` —— 字符串开头。
  - `(?=.*[a-z])` —— 前瞻断言，确保至少包含一个小写字母。
  - `(?=.*[A-Z])` —— 前瞻断言，确保至少包含一个大写字母。
  - `(?=.*\d)` —— 前瞻断言，确保至少包含一个数字。
  - `.{8,}` —— 任意字符至少 8 个。
  - `$` —— 字符串结尾。

**类比**：战斗记录分析——你打完一场 boss 战想看「我总共造成了多少伤害」。正则帮你把散落在 500 行战斗日志里的伤害数字一键提取。密码强度检查——冒险者公会注册时要求「密码至少 8 位、包含大小写和数字」，正则可以一次检查所有条件。

### 12.5 正则表达式（下）——文本清洗与任务解析

正则不仅能提取数据，还能清洗文本、替换格式标记。

```python
# 清洗 NPC 对话文本（去掉标记）
npc_text = "<anger>你竟敢闯入我的领地！</anger><normal>不过...如果你帮我找一样东西...</normal>"
clean = re.sub(r"<[^>]*>", "", npc_text)
# → "你竟敢闯入我的领地！不过...如果你帮我找一样东西..."

# 提取任务描述中的数字
quest = "收集 5 个狼牙、3 张熊皮、10 株草药"
counts = re.findall(r"(\d+)\s*[个张株]", quest)
# → ["5", "3", "10"]
```

**逐行解读：**

- `npc_text = "<anger>...<normal>..."` —— NPC 对话文本带情绪标签（`<anger>`=愤怒，`<normal>`=正常）。玩家只需要纯文字。
- `re.sub(r"<[^>]*>", "", npc_text)` —— `sub()` 是替换函数，三个参数：正则模式、替换成什么、目标字符串。
  - `<` —— 匹配左尖括号。
  - `[^>]*` —— `[^>]` 匹配任意非 `>` 的字符，`*` 表示零个或多个。合起来匹配尖括号内的所有内容。
  - `>` —— 匹配右尖括号。
  - `""` —— 用空字符串替换（即删除）。
  - 效果：删除所有 HTML/XML 风格的标签。
- `quest = "收集 5 个狼牙、3 张熊皮、10 株草药"` —— 任务描述文本。
- `re.findall(r"(\d+)\s*[个张株]", quest)` —— 提取任务需求数量：
  - `(\d+)` —— 捕获一个或多个数字。
  - `\s*` —— 零个或多个空白字符（数字和量词之间可能有空格）。
  - `[个张株]` —— 字符类，匹配「个」「张」「株」中任意一个。
  - 效果：提取出 `["5", "3", "10"]`。

**类比**：NPC 对话文本带了很多情绪标记（`<anger>`），你只需要纯文字显示给玩家。正则帮你洗掉标记。任务描述里提取「要几个什么」——交给 AI 写正则就对了。

**正则常用模式速查**：

| 模式     | 含义                  | 示例匹配                    |
| -------- | --------------------- | --------------------------- |
| `\d`     | 一个数字              | `"3"`                       |
| `\d+`    | 一个或多个数字        | `"145"`                     |
| `\w`     | 字母、数字、下划线    | `"A"`, `"3"`, `"_"`         |
| `\s`     | 空白字符              | 空格、Tab                   |
| `.`      | 任意字符（除换行）    | `"a"`, `"!"`                |
| `*`      | 零次或多次            | `\d*` 匹配 `""` 或 `"123"`  |
| `+`      | 一次或多次            | `\d+` 匹配 `"3"` 或 `"123"` |
| `[abc]`  | a、b、c 中任意一个    | `"a"` 或 `"b"`              |
| `[^abc]` | 除 a、b、c 外任意字符 | `"x"`, `"!"`                |
| `()`     | 捕获分组              | 提取括号内的内容            |
| `^`      | 字符串开头            |                             |
| `$`      | 字符串结尾            |                             |

### 挑战一：修复混乱的 NPC 对话

字符串妖精扔过来一段混乱的对话文本，满是首尾空格和重复的标点：
