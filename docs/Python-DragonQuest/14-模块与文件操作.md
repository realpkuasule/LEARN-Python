### 14.1 import——使用别人的代码

在 RPG 游戏里，你不会手动计算每一次暴击的随机数。你需要一个骰子系统——而 Python 内置的 `random` 模块就是那个骰子。你扭一下（`import random`），它帮你丢。

```python
import random

# 暴击率 15% → 随机判定
is_critical = random.random() < 0.15   # 15% 概率暴击

# 随机遇敌
monsters = ["史莱姆", "哥布林", "骷髅兵", "暗影骑士"]
encounter = random.choice(monsters)
print(f"遭遇了 {encounter}！")
```

**逐行解读**：

- `import random` —— 告诉 Python「我要用随机数功能」。这好比你在冒险者公会说「我要借一副骰子」
- `random.random()` —— 生成一个 0 到 1 之间的随机小数（比如 0.732）。`< 0.15` 意味着 15% 的概率为 `True`
- `is_critical` —— `True` 或 `False`，决定这刀是不是暴击
- `random.choice(monsters)` —— 从怪物名单里随机抽一个。`choice` 就是「从池子里随机选一个」的意思
- 每次运行代码，`encounter` 的值都可能不一样——这才是真正的随机遇敌

除了 `random`，数学计算也有现成的模块：

```python
import math
import random

# 伤害波动：基础伤害的 0.9 到 1.1 倍
base_damage = 25   # 前面章节算出来的基础伤害（必须先定义！）
final_damage = math.floor(base_damage * random.uniform(0.9, 1.1))
print(f"最终伤害：{final_damage}")
```

**逐行解读**：

- `base_damage = 25` —— **必须先定义**基础伤害值，后面的计算才能使用它
- `random.uniform(0.9, 1.1)` —— 在 0.9 到 1.1 之间随机取一个小数。`uniform` 是「均匀分布」的意思——区间内每个值被抽到的概率一样
- `base_damage * random.uniform(0.9, 1.1)` —— 基础伤害乘以一个 0.9~1.1 的随机倍率。比如 25 × 1.03 = 25.75
- `math.floor(...)` —— 向下取整。25.75 → 25。避免出现「造成了 25.75 点伤害」这种不合理输出（HP 只有整数——第 3 课说的）

**类比**：游戏内置的骰子系统（`random`）、伤害计算公式（`math`）、存档格式（`json`）。这些都是 Python 自带的功能模块——你 `import` 一下就能用，不用自己从零写随机数生成器。

JSON 模块也是同样的道理——用现成的，不自己造：

```python
import json

# 存档——把勇者数据转成 JSON 格式
hero = {"name": "勇者刘老三", "hp": 320, "atk": 65}   # hero 必须先定义！
save_data = json.dumps(hero, ensure_ascii=False)
print(save_data)
# → {"name": "勇者刘老三", "hp": 320, "atk": 65}
```

**逐行解读**：

- `hero = {...}` —— **必须先创建勇者字典**，`json.dumps` 才能把它转成字符串
- `json.dumps(hero, ensure_ascii=False)` —— 把 Python 字典转成 JSON 格式的字符串。`ensure_ascii=False` 确保中文字符正常显示（不写这个参数的话，「勇者刘老三」会变成 `勇者刘老三`）
- `dumps` —— dump string，把数据「倾倒」成字符串。还有一个 `dump`（没有 s）是直接「倾倒」到文件里，下面就会用到

### 14.2 自己的模块与 `__name__`

当你的游戏代码越来越多——怪物数据、战斗系统、存档系统、道具系统——全部写在一个文件里？那会是一个几千行的巨兽，改一个道具属性就要在代码海洋里捞针。

正确的做法：分文件，一个系统一个模块。

```python
# monsters.py —— 怪物数据模块
SLIME = {"name": "史莱姆", "hp": 30, "atk": 8, "def": 2}
GOBLIN = {"name": "哥布林", "hp": 50, "atk": 12, "def": 5}

# battle.py —— 战斗系统模块
from monsters import SLIME, GOBLIN

def attack(attacker, defender):
    damage = attacker["atk"] - defender["def"]
    if damage < 1:
        damage = 1
    defender["hp"] -= damage
    print(f"{attacker['name']} 攻击了 {defender['name']}！造成 {damage} 点伤害")

# main.py —— 游戏入口模块
from battle import attack
from monsters import SLIME, GOBLIN

if __name__ == "__main__":
    print("=== 勇者斗恶龙 ===")
    print("1. 新游戏  2. 读档  3. 退出")
    # main_menu()  这里调用主菜单函数
```

**逐行解读**：

- `monsters.py` —— 抽出一个专门的文件存放所有怪物的属性数据。改怪物属性时只改这一个文件
- `from monsters import SLIME, GOBLIN` —— 从 `monsters.py` 里取两个怪物数据。`from ... import` = 「从那个箱子里拿这两样东西」
- `battle.py` —— 战斗系统独立成模块。它从 `monsters` 模块导入怪物数据
- `main.py` —— 程序入口。它从 `battle` 和 `monsters` 两个模块导入需要的函数和数据
- `if __name__ == "__main__":` —— 这是一个关键的判断：**「这个文件是直接运行的，还是被别人 import 的？」** 如果直接运行 `main.py`，`__name__` 的值就是 `"__main__"`，条件成立，执行菜单代码。如果别人 `import main`，`__name__` 的值是 `"main"`（模块名），条件不成立，不会自动启动菜单

**类比**：游戏引擎的模块化开发——怪物数据一个文件夹、战斗系统一个文件夹、存档系统一个文件夹。每个团队各开发各的，互不干扰。`if __name__ == "__main__"` = 「双击运行这个文件就启动游戏，其他文件 import 它的时候不会」。

### 14.3 文件读写操作——存档与读档

这是本课最核心的内容。存档和读档是 RPG 的生命线——代码很简单，意义很重大。

```python
import json

def save_game(hero, filename="save.json"):
    """存档！把勇者的全部数据保存到文件"""
    save_data = {
        "hero": hero,
        "timestamp": "2026-04-28 19:23",
        "location": "恶龙山脉·入口",
    }
    with open(filename, "w", encoding="utf-8") as f:
        json.dump(save_data, f, ensure_ascii=False, indent=2)
    print("存档成功！")

def load_game(filename="save.json"):
    """读档！从文件读取勇者的数据"""
    with open(filename, "r", encoding="utf-8") as f:
        save_data = json.load(f)
    print(f"读取存档——{save_data['timestamp']}，地点：{save_data['location']}")
    return save_data["hero"]
```

**逐行解读**：

- `save_game(hero, filename="save.json")` —— 存档函数。接收勇者数据，默认存到 `save.json`
- `save_data = {...}` —— 构造存档对象。不仅存勇者数据，还存时间戳和当前位置——跟真正的 RPG 存档一样
- `with open(filename, "w", encoding="utf-8") as f:` —— 以**写入**模式（`"w"`）打开文件。`with` 语句保证代码块执行完后文件自动关闭（不用手动 `f.close()`）。`encoding="utf-8"` 是第 13 课的铁律——防止乱码
- `json.dump(save_data, f, ensure_ascii=False, indent=2)` —— 把 `save_data` 字典写入文件 `f`。注意这里是 `dump`（没有 s），直接写入文件对象。`indent=2` 让 JSON 格式化缩进，人类也能读
- `load_game(filename="save.json")` —— 读档函数。`"r"` 模式打开文件，`json.load(f)` 把文件里的 JSON 读出来转回 Python 字典
- `return save_data["hero"]` —— 把存档里存的勇者数据返回给调用者

**类比**：RPG 的教堂存档点——你跪在教堂祈祷（`save_game`），冒险经历被刻入石碑。下次打开游戏，从石碑读取（`load_game`），继续冒险。没有存档，一切白打——这就是为什么每个老玩家都会反复说「存档了吗？」

### 14.4 路径处理——跨平台兼容

Windows 的路径是 `saves\slot_1\save.json`（反斜杠），Mac 和 Linux 是 `saves/slot_1/save.json`（正斜杠）。如果你在代码里手写路径——换一台电脑可能就报错了。

```python
import os

# 不要手写路径！Windows 和 Mac 分隔符不一样
save_dir = os.path.join("saves", "slot_1", "save.json")
print(save_dir)
# Windows → saves\slot_1\save.json
# Mac    → saves/slot_1/save.json

# 存档存在吗？——先检查再读档
if os.path.exists(save_dir):
    print("存档存在，读取中...")
    # hero = load_game(save_dir)
else:
    print("没有存档，开始新游戏！")
    # hero = create_new_hero()
```

**逐行解读**：

- `import os` —— 操作系统接口模块（Operating System）。提供路径处理、文件检查等功能
- `os.path.join("saves", "slot_1", "save.json")` —— 用系统正确的分隔符拼接路径。你不需要知道当前系统用 `/` 还是 `\`，`os.path.join` 自动处理
- `os.path.exists(save_dir)` —— 检查文件或目录是否存在。返回 `True` 或 `False`
- 如果存档存在，就读取；如果不存在，就创建新勇者——这是每个 RPG 启动时做的第一件事

**类比**：插卡带——存档文件在「saves/slot_1」里。`os.path.exists` = 「摸摸卡槽里有没有卡带」。`os.path.join` = 「不管你用的是什么机器，我都能找到那个卡槽的正确位置」。

完整的启动流程示例：

```python
import os
import json

def init_game():
    """游戏初始化——检查存档"""
    save_path = os.path.join("saves", "slot_1", "save.json")

    if os.path.exists(save_path):
        with open(save_path, "r", encoding="utf-8") as f:
            data = json.load(f)
        print(f"欢迎回来，{data['hero']['name']}！")
        return data["hero"]
    else:
        print("没有存档，创建新勇者！")
        hero = {"name": "勇者刘老三", "hp": 100, "max_hp": 100,
                "mp": 50, "max_mp": 50, "atk": 15, "def": 5,
                "level": 1, "gold": 0}
        return hero

# 启动游戏
hero = init_game()
print(f"当前状态：Lv.{hero['level']} HP:{hero['hp']}/{hero['max_hp']} ATK:{hero['atk']}")
```

**逐行解读**：

- `init_game()` —— 游戏初始化的统一入口。先检查有没有存档，有就读，没有就建
- `save_path = os.path.join("saves", "slot_1", "save.json")` —— 拼接存档文件的完整路径
- `os.path.exists(save_path)` —— 判断存档文件是否真的存在于磁盘上
- 有存档：用 `json.load(f)` 读取并重建勇者数据。显示「欢迎回来」
- 没存档：创建一个全新的勇者角色卡，所有属性都是初始值
- 最后返回勇者数据，游戏正式开始

**场景一：你需要给游戏加上多存档槽位功能**

Prompt 示例：

> 我的文字 RPG 目前只有一个存档文件 `save.json`。请帮我升级存档系统，支持 3 个存档槽位（`save_1.json`、`save_2.json`、`save_3.json`）。需要实现以下功能：
>
> 1. `save_game(hero, slot)` —— 存到指定槽位
> 2. `load_game(slot)` —— 从指定槽位读档
> 3. `list_saves()` —— 列出所有存档（显示槽位、角色名、等级、时间、地点）
>
> 使用 `os.path.exists` 检查存档是否存在，`json.dump/json.load` 读写，所有文件操作指定 `encoding='utf-8'`。

**场景二：你想把项目拆分成多个模块文件**

Prompt 示例：

> 我的文字 RPG 现在所有代码都在一个 `main.py` 里，太长了。请帮我按功能拆分成以下模块文件：
>
> - `monsters.py` —— 怪物数据（每个怪物一个字典）
> - `battle.py` —— 战斗系统（攻击、计算伤害、战斗循环）
> - `items.py` —— 道具系统（药水、装备、效果表）
> - `save.py` —— 存档读档（使用 `json` 和 `os.path`）
> - `main.py` —— 入口文件（使用 `if __name__ == "__main__"` 启动游戏）
>
> 每个模块之间通过 `from xxx import yyy` 互相引用，保证代码可运行。
