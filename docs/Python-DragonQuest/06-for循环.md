# 第六课：for 循环

> 遍历背包物品、清点战利品、给队伍全员回血——全是 for 循环。for 帮你把"对每一个做一遍"写成一行代码。

## 冒险引入

勇者刘老三一剑劈翻了最后一只史莱姆。战斗结束，地上散落着亮晶晶的战利品：铜剑一把、回复药两瓶、魔法戒指一枚、金币袋一个。

"又是丰收的一天！"刘老三蹲下来，一样一样往背包里装。第一件，铜剑——放进背包。第二件，回复药——放进背包。第三件，又是回复药——再放。第四件，魔法戒指——稀有货！第五件，金币袋——今天的酒钱有了。

装了五件，他还有耐心。但如果打完哥布林部落，掉落了二十件战利品呢？难道要写二十遍 `print("获得了：xx！")` ？更麻烦的是——队伍里四个人，打完 boss 战牧师得给每个人回血。"勇者回满、法师回满、盗贼回满、牧师自己回满"——对每个人做一模一样的事，只是对象不同。程序里，这叫 **for 循环**。

## 核心讲解

### 6.1 for 循环基础——"每件战利品念一遍名字"

每次战斗结束，打开背包一个个看掉了什么。掉了几件就看几件——名单有多长看多长。

```python
# 战斗结束，清点战利品
掉落列表 = ["铜剑", "回复药", "回复药", "魔法戒指", "金币袋"]

for 物品 in 掉落列表:
    print(f"获得了：{物品}！")
```

**逐行解读**：

- 第 2 行：`掉落列表` 是一个列表——战利品清单
- 第 4 行：`for 物品 in 掉落列表`——把 `掉落列表` 里的每个东西依次取出来，临时贴个标签叫 `物品`
- 第 5 行：对每一个 `物品` 执行 `print`——第一轮 物品 = "铜剑"，打印；第二轮 物品 = "回复药"，打印……直到清单结束
- 循环自动停止——名单有多长就跑多少轮，不用你数

**类比**：收银员扫商品。你把购物篮放上台，他一个一个扫码——"滴，铜剑"、"滴，回复药"、"滴，魔法戒指"。全部扫完自动停。`for` 就是那个扫码枪——你给它一个清单，它逐个处理。

```python
# 给队伍所有成员回满血
队伍 = ["勇者", "法师", "牧师", "盗贼"]
for 成员 in 队伍:
    print(f"正在为 {成员} 恢复 hp...")
    # heal_to_full(成员)  ← 第 8 课会把这个写成真正的函数
    print(f"{成员} 的 hp 已恢复！")
```

**逐行解读**：

- `队伍` 是队伍名单——四个人，四种职业
- `for 成员 in 队伍`——每个成员依次成为 `成员`，执行同样的"回血"动作
- 牧师给队伍全员加血——对每个人做一遍"治疗术"

### 6.2 range() 函数——"第 1 到第 10 回合"

战斗有回合数。第 1 回合、第 2 回合……直到战斗结束。`range()` 就是回合计数器。

```python
# 战斗回合计数器
for 回合 in range(1, 11):    # 第 1 到第 10 回合
    print(f"=== 第 {回合} 回合 ===")
    # 这里调用战斗逻辑（第 8 课会封装成函数）
```

**逐行解读**：

- `range(1, 11)`——从 1 开始，到 11 之前停止（也就是 1、2、3……10）
- `range(10)` 等价于 `range(0, 10)`——从 0 开始计数 10 个（0 到 9）
- `range(1, 11, 2)`——加第三个参数"步长"，输出 1、3、5、7、9（每 2 步一跳）

**RPG 场景**：你知道这场 boss 战最多 30 回合（狂暴倒计时）。`range` 就是那个显示在屏幕右上角的回合计数器。

```python
# 每 3 回合触发一次环境效果（比如火山地形灼烧）
for 回合 in range(1, 31):
    if 回合 % 3 == 0:        # 第 3、6、9... 回合——能被 3 整除
        print(f"第 {回合} 回合：地形效果触发！全体受到灼烧伤害！")
    else:
        print(f"第 {回合} 回合：正常战斗")
```

**逐行解读**：

- `回合 % 3` 是"回合 除以 3 的余数"——如果余数是 0，说明刚好是 3 的倍数
- 第 3、6、9、12……回合触发环境效果
- `range + 取余` = 定时触发器——boss 每 3 回合放一次 AOE，就是靠这个实现的

### 6.3 break、continue 与循环嵌套——"找到传说之剑就别翻了"

有时候不需要遍历完——找到目标就停。有时候某些东西要跳过不处理。

```python
# 在背包里找「传说之剑」
背包 = ["木剑", "铜盾", "回复药", "传说之剑", "破布"]

for 物品 in 背包:
    if 物品 == "传说之剑":
        print("找到了！装备上！")
        break          # 找到了，后面的不用翻了
    print(f"{物品}...不是这个")
```

**逐行解读**：

- 第 4 行：`for 物品 in 背包`——逐个翻背包
- 第 5-7 行：翻到"传说之剑"！打印提示，然后 `break` 退出循环
- 第 8 行：还没找到——打印"不是这个"，继续翻下一格
- 执行顺序：木剑→不是、铜盾→不是、回复药→不是、传说之剑→找到了+break，破布根本没被翻到

```python
# 批量使用道具——跳过「不可使用」的装备
背包 = ["木剑", "回复药", "铜盾", "解毒草", "破布", "魔力药"]

for 物品 in 背包:
    if 物品 in ["木剑", "铜盾", "破布"]:
        continue       # 这是装备不是消耗品，跳过不处理
    print(f"使用了 {物品}！")
    # use_物品(物品)  ← 第 8 课会把这个写成函数
```

**逐行解读**：

- 第 4 行：检查当前物品是不是"不可使用"的装备
- 第 5 行：`continue`——跳过本次循环，直接进入下一件物品
- 第 6-7 行：只有能用的（回复药、解毒草、魔力药）才会执行到这里
- break 是"彻底不翻了"，continue 是"这件跳过，继续翻下一件"

**类比**：翻背包找钥匙——一个一个翻，找到了就拿起来（break）。翻到装备不消耗（continue），翻到药水才喝。

```python
# 嵌套循环：检查迷宫每个房间的每块地板
房间列表 = ["入口", "走廊", "宝库", "王座"]

for 房间 in 房间列表:
    print(f"\n进入【{房间}】")
    for 地板 in range(1, 6):         # 每个房间 5 块地板
        if 房间 == "宝库" and 地板 == 3:
            print(f"  地板[{地板}]：发现陷阱！小心！")
        else:
            print(f"  地板[{地板}]：安全")
```

**逐行解读**：

- 外层 `for 房间 in 房间列表`——逐个进入房间
- 内层 `for 地板 in range(1, 6)`——在每个房间里逐块检查地板
- 嵌套循环的执行顺序：房间"入口"→地板 1 2 3 4 5→房间"走廊"→地板 1 2 3 4 5→...
- 总共执行 4 × 5 = 20 次循环体

**类比**：就像学校检查仪容仪表——从第一排到最后一排，每一排从左到右挨个检查。

## AI 协作指南

### 场景一：让 AI 帮你写战利品统计

打完一场大战，背包里东西太多——你想知道每种物品各有多少个。

> "我有一个战利品列表 `掉落列表 = ['回复药', '铜剑', '回复药', '魔法戒指', '回复药', '铜剑', '金币袋']`。帮我用 for 循环统计每种物品的数量，输出格式为 '回复药 x3，铜剑 x2，魔法戒指 x1，金币袋 x1'。不要用任何还没学过的数据结构（比如字典），只用 for + if + 列表。"

AI 会给你一个使用 `掉落列表.count()` 或手动计数的方法。你重点关注它用的循环结构——哪里该 break、哪里该 continue。

### 场景二：让 AI 优化你的循环代码

你写了一段遍历背包的代码，感觉有点啰嗦。贴给 AI：

> "下面这段代码遍历背包找药水，找到了就用。帮我优化——用 for + break 改写，找到第一瓶回复药就停止遍历。并解释 break 为什么能提高效率。"

```python
背包 = ["木剑", "解毒草", "回复药", "回复药", "铜盾"]
for 物品 in 背包:
    if 物品 == "回复药":
        print("使用回复药！")
```

AI 会告诉你：上面的代码会把两瓶回复药都用了——可能不是你想要的。应该找到第一瓶就 `break`。
