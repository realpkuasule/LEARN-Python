# 第五课：条件判断

> 战斗逻辑 = 一连串的"如果...就..."。hp 归零了吗？暴击了吗？敌人有属性弱点吗？每个判断都是一个 if。

---

## 冒险引入

勇者刘老三提着铁剑走到荒野深处。前方草丛窸窣作响——一只史莱姆跳了出来！

"好，准备战斗！"刘老三握紧剑柄。战斗系统在他脑中飞速运转：我的攻击力 15，史莱姆防御力 5，一击打多少血？伤害 = 15 - 5 = 10 点。如果史莱姆是冰属性、我是火属性——伤害翻倍！如果它防御比我的攻击还高？那就保底 0 点伤害，不能倒过来给它加血吧？

但刘老三很快就发现一个问题——战斗里的判断不是那么简单的。有些判断需要"同时满足好几个条件"——比如面对冰属性的敌人，你需要自己武器是火属性"火克冰"，而且这一刀还要触发"暴击"，两个条件都满足了，伤害才能"翻倍再翻倍"，变成原来的 4 倍伤害。这就是"同时满足"的判断。

还有些判断是"依次检查，找到匹配的就用"——你看到一只怪物，先看看"它是史莱姆吗？如果是，那就用最简单的攻击，史莱姆太弱了没必要用大招"。如果不是史莱姆，再看"它是恶龙吗？如果是恶龙，那就要用针对性的战术，恶龙会放龙息，得小心应对"。如果既不是史莱姆也不是恶龙，那才按"普通怪物"的打法处理。这就是"依次筛选"的判断。

这两种判断方式在战斗中每天都在发生。程序里的条件判断，就是把你脑子里这些"如果...就..."的想法，精确地写成代码。"如果敌人是冰属性而且我触发暴击，伤害就翻倍"；"如果敌人是史莱姆，就用简单攻击；如果敌人是恶龙，就用针对性战术；如果不是这两种，就用普通打法"。每个战斗决策都是一个条件判断。

## 核心讲解

### 5.1 if 语句基础——"如果伤害太低怎么办？"

每个回合你都在判断："我的攻击 > 对方防御吗？"大于就造成伤害，不够就"叮！攻击被弹开了"。程序只是把你脑中的判断写下来。

```python
# 勇者攻击怪物——统一的伤害公式
勇者攻击力 = 15
怪物防御力 = 5
怪物hp = 30   # 怪物初始血量
对怪物伤害 = 勇者攻击力 - 怪物防御力

if 对怪物伤害 <= 0:
    print("攻击无效！怪物防御太高了！")
    对怪物伤害 = 0   # 保底 0 点伤害——不能倒过来加血
else:
    print(f"造成了 {对怪物伤害} 点伤害！")

怪物hp = 怪物hp - 对怪物伤害   # → 30 - 10 = 20
print(f"怪物剩余 hp：{怪物hp}")
```

**逐行解读**：

- 第 1 行：给 `对怪物伤害` 贴上计算结果（15 - 5 = 10）
- 第 2 行：`if 对怪物伤害 <= 0`——"砍不动吗？"如果防御比攻击还高，伤害就是负数或零
- 第 3-4 行：真砍不动！打印提示，保底 0 点伤害（游戏规则：不能攻击反而给怪物加血）
- 第 5-6 行：砍得动！正常打出伤害
- 第 8 行：无论哪个分支，怪物血量都要扣掉——注意这行在 if/else 外面

**类比**：每个回合你都在脑中判断——这一剑砍得动吗？砍得动就扣血，砍不动就弹刀。程序只是把这个脑内判断写成了 `if 对怪物伤害 <= 0`。

### 5.2 if/elif/else 与嵌套——"火克冰，同属性减半"

真实战斗里不止一种情况。火克冰、冰克火、同属性减半——这需要一连串的判断。

```python
# 元素克制判断
勇者属性 = "火"
怪物属性 = "冰"
对怪物伤害 = 20   # 基础伤害（前面已经算好了）

if 勇者属性 == "火" and 怪物属性 == "冰":
    对怪物伤害 = 对怪物伤害 * 2    # 火克冰，双倍伤害！
    print("火属性克制！伤害翻倍！")
elif 勇者属性 == 怪物属性:
    对怪物伤害 = 对怪物伤害 // 2   # 同属性，伤害减半
    print("同属性，效果不佳...")
else:
    print("普通攻击")       # 没有克制关系，正常伤害

print(f"最终伤害：{对怪物伤害}")
```

**逐行解读**：

- 第 1-2 行：先确定双方属性（正常情况下这些来自角色面板和怪物图鉴）
- 第 4 行：`if 勇者属性 == "火" and 怪物属性 == "冰"`——两个条件同时满足才触发（火 AND 对方是冰）
- 第 5-6 行：条件成立！伤害翻倍，打印克制提示
- 第 7 行：`elif`——"如果上一个不成立，那试试这个"。同属性，伤害减半
- 第 9 行：`else`——以上都不成立，普通伤害
- 第 12 行：最终伤害上线，无论走哪条路这一行都会执行

**关键区别**：`elif` 和一连串 `if` 不同。`elif` 是"选一个执行"，第一个命中就跳过后面。如果你写成三个独立的 `if`，可能每个都判断一遍——浪费且可能出错。

```python
# 判断怪物类型——不同怪物行为不同（嵌套判断）
怪物类型 = "史莱姆"
hp = 30

if 怪物类型 == "史莱姆":
    print("史莱姆很弱，直接攻击即可！")
    if hp < 10:
        print("  它快不行了，最后一击！")
elif 怪物类型 == "恶龙":
    print("注意！恶龙每3回合会释放龙息！")
    if hp > 200:
        print("  恶龙状态极佳，建议先防御")
else:
    print("普通的怪物，正常战斗")
```

**逐行解读**：

- 第 5 行：先判断是不是史莱姆（第一层）
- 第 6 行：是史莱姆！再判断血量是不是低于 10（嵌套的第二层）
- 第 7 行：两层都满足——"它快不行了"
- 第 8 行：不是史莱姆，试试是不是恶龙（同级 elif）
- 第 10 行：是恶龙！再判断血量（嵌套判断）

**类比**：勇者斗恶龙的属性克制表：火克冰、冰克风、风克火。程序判断：我方什么属性？敌人什么属性？按照克制表翻倍或减半。

### 5.3 条件表达式与实战——"暴击了一行搞定"

有时候 if/else 整个结构太啰嗦——只想根据一个条件切换两个值：

```python
# 暴击判定——一行搞定（条件表达式）
是暴击 = True
对怪物基础伤害 = 25
对怪物最终伤害 = 对怪物基础伤害 * 2 if 是暴击 else 对怪物基础伤害
print(f"最终伤害：{对怪物最终伤害}")   # → 50（暴击了！）
```

**逐行解读**：

- 这行读起来像人话："最终伤害 = 基础伤害乘 2（如果暴击了）否则就是基础伤害"
- `A if 条件 else B` 是 Python 的三元表达式——当条件为 True 取 A，False 取 B
- 适合简单的"二选一"，复杂的判断还是老老实实写 if/elif/else

```python
# 战斗中的完整决策链
勇者hp = 85
怪物hp = 20
勇者攻击力 = 15
有回复药 = True

# 第一步：判断血量状态
if 勇者hp < 30:
    if 有回复药:
        print("血量危险！使用回复药！")
        # 勇者hp += 30  ← 实际上应该调用 use_potion()
    else:
        print("血量危险且无药！考虑逃跑！")
elif 勇者hp >= 80:
    print("血量充足，放心输出！")

# 第二步：判断是否该全力输出
if 怪物hp < 30:
    print("怪物残血，全力攻击！")
    对怪物最终伤害 = (勇者攻击力 * 2) if 怪物hp < 15 else 勇者攻击力
    print(f"造成 {对怪物最终伤害} 点伤害")
```

**逐行解读**：

- 第 8-12 行：先判断自己的血量（危险 → 用药 / 逃 → 健康 → 输出）
- 第 14-17 行：再判断怪物血量（残血 → 考虑暴击追一刀）
- 第 16 行：又一个条件表达式——怪物血量低于 15 时翻倍，否则普通伤害

**核心陷阱**：`=` 是贴标签（赋值），`==` 是"一样吗"（比较）。"勇者刘老三"写在铭牌上（`=`）和"这是勇者刘老三吗"（`==`）是两件完全不同的事。搞混了这个，AI 的代码会出神秘 bug——`if 勇者hp = 0` 会报错。

## AI 协作指南

### 场景一：让 AI 帮你写属性克制表

你可以这样问 AI 魔法书：

> "我有一段伤害计算代码，需要加入属性克制逻辑。规则是：火克冰克风克火（循环克制），克制时伤害翻倍，被克制时伤害减半，同属性不变。帮我写一个 `计算属性伤害(攻击属性, 防御属性, 对怪物基础伤害)` 函数，返回最终伤害。"

AI 会帮你写出完整的 if/elif 链。你只需要告诉它规则——它负责把规则翻译成代码。

### 场景二：让 AI 帮你排错

如果战斗伤害总是不对，把代码贴给 AI 说：

> "下面这段战斗代码里，怪物防御 20、我攻击 15，但伤害还是 15 而不是保底的 1。帮我找 bug，并解释为什么 if 条件没生效。"

```python
对怪物伤害 = 勇者攻击力 - 怪物防御力
if 对怪物伤害 = 0:      # ← AI 会指出：这是赋值，不是比较！
    对怪物伤害 = 1
```

AI 不仅能指出 `=` 应该是 `==`，还会解释为什么。因为 `if 对怪物伤害 = 0` 语法上是错的。
