## 核心讲解

### 变量——给数据贴标签

```python
hp = 100        # 给数字 100 贴了标签「hp」
mp = 50         # 给数字 50 贴了标签「mp」
name = "勇者张三"  # 给文字贴了标签「name」

# 挨了怪物一下
hp = hp - 15    # 把标签 hp 从 100 撕下来，贴到 85 上
```

**逐行解读**：

- `hp = 100` —— 创建了一个整数 `100`，然后给这个数据贴上标签 `hp`。以后说 `hp`，就是在说这个数据。
- `mp = 50` —— 同理，把标签 `mp` 贴到 `50` 上。
- `name = "勇者张三"` —— 标签 `name` 贴在了一串文字上。引号表示这是文字，不是变量名。
- `hp = hp - 15` —— 先把右边的 `hp`（当前值 100）减 15 得到 85，然后把标签 `hp` 从 100 上撕下来，贴到新值 85 上。

**重要思维**：不要说「变量是装数据的盒子」。盒子装东西就固定了。标签可以随时撕下来换贴。这不是咬文嚼字——你在第 8 课学函数时如果还觉得变量是盒子，作用域那块会非常痛苦。现在就把思维扭过来：**变量 = 标签，不是盒子。**

**冒险类比**：你的冒险者铭牌。铭牌上写着「勇者张三」，不是你本人变成了一张纸——铭牌只是贴在身上的标签。受伤了，HP 标签从「100」换成「85」，人还是同一个人。

### 数字类型——整数与小数

```python
hp = 100            # int（整数）——生命值，只有整点
crit_rate = 0.15    # float（小数）——暴击率 15%

# 整除：100 金币，恢复药 30 一瓶，最多买几瓶？
max_potions = 100 // 30   # → 3（整除，扔掉余数）

# 普通除法：属性克制伤害 ×1.5
base_damage = 10
boosted_damage = base_damage * 1.5   # → 15.0（结果是小数）
```

**逐行解读**：

- `hp = 100` —— 100 是整数（`int`）。HP 只能是整数——没有「半滴血」。
- `crit_rate = 0.15` —— 0.15 是小数（`float`）。暴击率和属性倍率可以是小数。
- `100 // 30` —— `//` 是整除运算符，结果只留整数部分。100 金币 ÷ 30 = 3 瓶，余下的 10 金币不够再买一瓶。
- `base_damage * 1.5` —— 普通除法 `/` 会产生小数结果。10 × 1.5 = 15.0。

| 类型            | 冒险例子           | 说明                                  |
| --------------- | ------------------ | ------------------------------------- |
| `int`（整数）   | `hp = 100`         | 生命值，只有整点                      |
| `float`（小数） | `crit_rate = 0.15` | 暴击率 15%                            |
| `//` 整除       | `100 // 30 = 3`    | 100 金币，恢复药 30 一瓶，最多买 3 瓶 |
| `/` 普通除      | `damage * 1.5`     | 属性克制——伤害 ×1.5                   |

**冒险思维**：你不需要背类型名，但需要知道数字能有两种形状存在——整的（`int`，比如等级、HP、金币数）和带小数的（`float`，比如暴击率、属性倍率）。

### 字符串类型——角色的名字与称号

```python
hero_name = "勇者张三"
title = "史莱姆克星"
full_title = hero_name + "·" + title   # → "勇者张三·史莱姆克星"

divider = "=" * 30   # → "=============================="
```

**逐行解读**：

- `hero_name = "勇者张三"` —— 字符串（`str`）用引号括起来。单引号 `'...'` 或双引号 `"..."` 都可以，但要配对。
- `title = "史莱姆克星"` —— 另一个字符串。干掉一只怪物 → 获得称号。
- `hero_name + "·" + title` —— 字符串用 `+` 拼接。注意中间的 `"·"` 也是字符串——一个字一个字粘起来。
- `"=" * 30` —— 字符串用 `*` 重复。`"="` 这个字符重复 30 次，等于画了一条分隔线。

**冒险类比**：称号系统。干掉一只怪物 → 获得称号 → 拼在名字后面。杂货铺招牌上画分隔线，`"=" * 30` 就是画线。

### 布尔类型与 None——「是/否」与「还没有」

```python
is_alive = True           # 还活着
has_weapon = False        # 手无寸铁
is_stunned = False        # 没被眩晕

quest_item = None   # 任务物品还没拿到——不是「没有」，是「还没」
```

**逐行解读**：

- `is_alive = True` —— 布尔类型（`bool`），只有 `True` 和 `False` 两个值。注意大写开头。
- `has_weapon = False` —— `False` 表示「否」。勇者出门忘带武器。
- `is_stunned = False` —— 眩晕状态。这些布尔值后面会在 `if` 语句中用（第 5 课）。
- `quest_item = None` —— `None` 是 Python 里表示「空」的特殊值。不是「这把剑不存在」，是「任务物品还没拿到手」。

**冒险类比**：勇者状态栏——活着（True）/ 死了（False）、有武器（True）/ 空手（False）、被眩晕（True）/ 清醒（False）。`None` 是任务追踪——「还没拿到任务物品」，不是不存在，是还没触发。

## AI 协作指南

### 场景一：让 AI 帮你理解类型错误

当你写代码时把类型搞混了，AI 是最好的纠错伙伴：

> **Prompt 示例**：「我在写 Python 代码时遇到这个错误：`TypeError: can only concatenate str (not "int") to str`。我的代码是 `print("HP:" + 100)`。帮我解释为什么错，以及应该怎么改。」

AI 会告诉你：`"HP:"` 是字符串，`100` 是整数，不能直接用 `+` 拼接。应该用 f-string：`print(f"HP:{100}")` 或转成字符串：`print("HP:" + str(100))`。

### 场景二：用 AI 检查变量命名和设计

创建角色属性时，让 AI 帮你审查设计：

> **Prompt 示例**：「我正在用 Python 做一个 RPG 游戏的角色系统。目前我设计了这些变量：`hp = 100`、`mp = 50`、`atk = 15`、`def = 10`。请帮我检查：（1）有没有漏掉重要的属性？（2）`def` 作为变量名有没有问题？（3）还有哪些属性应该加进来？」

AI 会发现 `def` 是 Python 的关键字（定义函数用的），不能做变量名，建议改成 `def_stat`。还会建议你加上 `level`、`gold`、`exp`、`max_hp`、`max_mp` 等。

## 实战练习

### 练习1：改一行代码，看输出变化（最简单）

在 Python 交互模式中逐行输入：

```python
hero_name = "勇者张三"
title = "史莱姆克星"
full_title = hero_name + title
print(full_title)
```

输出是 `勇者张三史莱姆克星`——少了分隔符。现在改第三行，在中间加一个 `"·"`：

```python
full_title = hero_name + "·" + title
print(full_title)   # → "勇者张三·史莱姆克星"
```

再试试把 `title` 改成 `"恶龙杀手"`（终章的称号！），重新运行。验证：不改变拼接逻辑，只换数据，输出自动变了——这就是变量的威力。

### 练习2：加一个新功能——创建完整的角色卡（中等）

创建一个新文件 `hero_card.py`，写入完整角色卡：

```python
# === 勇者张三·完整角色卡 ===

# 基础属性
name = "勇者张三"
title = "史莱姆克星"
level = 1
hp = 100
max_hp = 100
mp = 50
max_mp = 50
atk = 15
def_stat = 10
gold = 500
exp = 0

# 状态
is_alive = True
has_weapon = True
quest_item = None   # 还没接任务

# 打印角色卡
full_name = name + "·" + title
print("=" * 30)
print("  " + full_name)
print("  Lv." + str(level))
print("  HP: " + str(hp) + "/" + str(max_hp))
print("  MP: " + str(mp) + "/" + str(max_mp))
print("  ATK: " + str(atk) + "  DEF: " + str(def_stat))
print("  金币: " + str(gold))
print("=" * 30)
```

运行看效果。你会发现在 `print` 里用 `+` 拼接需要频繁写 `str()`——很麻烦。没关系，第 4 课会学 f-string，一行搞定。现在先忍受「拼接的繁琐」，到时候你才明白 f-string 有多香。

### 练习3：结合 AI，让 AI 帮你扩展（综合）

把练习 2 的代码发给 AI：

> **Prompt 示例**：「这是我的 Python 角色卡代码：{粘贴代码}。请帮我做以下扩展：（1）加入 `crit_rate = 0.15`（暴击率）和 `dodge_rate = 0.05`（闪避率）；（2）在角色卡打印中加入这两项；（3）用 f-string 方式重写所有 print 语句（更简洁）；（4）角色击败一只史莱姆后，获得 30 经验值、50 金币，请写一段更新代码。」

AI 会输出扩展后的代码。把输出保存为 `hero_card_v2.py`，对比两版的 `print` 部分——你会立刻体会到 f-string 比 `+` 拼接清爽多少。
