## 冒险引入

勇者刘老三在洞穴深处遇到了哥布林守卫。双方对峙。

"来吧！"刘老三拔出铁剑。他知道 for 循环能数回合——但那需要提前知道打多少回合。可这场战斗，可能 3 回合结束（暴击两连斩），也可能拖到 20 回合（双方都在嗑药）。他不知道终点在哪。

哥布林先动了！一爪挥来——刘老三侧身闪过，反手一剑。哥布林吃痛后退，又扑上来。你一刀，我一刀，你一刀，我一刀……这就是回合制战斗的核心循环。不会在第一回合就结束，也不会永远打下去——它有一个明确的停止条件：**一方 hp 归零**。

"只要双方都还活着，战斗就继续。"这是 while 循环的领域。for 适合"遍历背包 10 件物品"，while 适合"打到一方倒下为止"。

## 核心讲解

### 7.1 while 语法——"打到一方倒下为止"

你玩任何一个回合制 RPG——战斗基本上都是"你一刀我一刀，一直重复到谁先倒下"。你不知道第几回合会结束，这种情况下应该用 while循环。while 条件 = "双方都还活着"。

```python
# 勇者和怪物的"属性卡"（用字典存储）
勇者 = {"hp": 100, "攻击力": 15, "名字": "勇者刘老三"}
怪物 = {"hp": 50, "攻击力": 8, "名字": "哥布林"}

# 战斗主循环！
while 勇者["hp"] > 0 and 怪物["hp"] > 0:
    print(f"\n刘老三 hp: {勇者['hp']} | {怪物['名字']} hp: {怪物['hp']}")

    行动 = input("1.攻击 2.魔法 3.道具 4.逃跑 > ")

    if 行动 == "1":
        伤害 = 勇者["攻击力"] - 怪物.get("def", 0)
        if 伤害 <= 0:
            伤害 = 0   # 保底 0 点伤害，避免造成“攻击给怪物加血”
        怪物["hp"] -= 伤害
        print(f"刘老三攻击！造成 {伤害} 点伤害")
    elif 行动 == "4":
        print("勇者逃跑了！")
        break

    # 怪物还活着的话就反击
    if 怪物["hp"] > 0:
        勇者["hp"] -= 怪物["攻击力"]
        print(f"{怪物['名字']} 反击！造成 {怪物['攻击力']} 点伤害")

# 战斗结束——判断结果
if 勇者["hp"] <= 0:
    print("刘老三倒下了...游戏结束")
elif 怪物["hp"] <= 0:
    print(f"击败了{怪物['名字']}！获得经验值！")
```

**逐行解读**：

- 第 1-2 行：用字典创建勇者和怪物的属性卡——`hp`、`攻击力`、`名字` 一目了然
- 第 5 行：`while 勇者["hp"] > 0 and 怪物["hp"] > 0`——这是战斗继续的条件。两个条件用 `and` 连接：勇者活着 **且** 怪物活着，只要有一个倒下就停止
- 第 6 行：每回合开头显示双方血量——用 f-string 嵌入字典取值
- 第 8 行：`input()` 等待玩家输入指令——拿到的是字符串 "1" 或 "2" 等
- 第 10-14 行：攻击分支——伤害 = 攻击力 - 防御力（怪物可能没有 def，用 `.get("def", 0)` 安全取值），伤害至少为 1
- 第 15-17 行：逃跑分支——打印提示，`break` 跳出循环
- 第 20-22 行：怪物反击——注意这里先判断 `怪物["hp"] > 0`，因为勇者可能已经打死了怪物，死掉的怪物不应该反击
- 第 25-28 行：循环结束后判断结果——谁先倒了？

**类比**：拳击比赛——双方上场，裁判喊"开始"。你一拳我一拳，直到有人倒下或举白旗（逃跑）。裁判不知道第几回合会结束，他只需要每回合看一眼"两人都站着吗？"——这就是 `while` 的条件检查。

### 7.2 死循环与循环控制——"boss 锁血了怎么办？"

写 while 时第一句就该问：**什么情况下会停？** 如果答案不明确，你的程序就会变成死循环。

```python
import time
# 危险！boss 锁血了——永远打不死（死循环示例）
boss = {"hp": 1000, "攻击力": 50, "名字": "恶龙"}

while boss["hp"] > 0:
    boss["hp"] -= 120
    print(f"对恶龙造成 120 点伤害！剩余 hp：{boss['hp']}")
    time.sleep(1)   # 暂停 1 秒
    # boss 有个被动：低于 100 血时自动回满
    if boss["hp"] < 100:
        boss["hp"] = 1000   # ← 死循环！永远打不死！
        print("恶龙发动「龙血复苏」！hp 回满了！")
        time.sleep(1)   # 暂停 1 秒
```

**逐行解读**：

- 第 4 行：循环条件——"boss 血量大于 0"
- 第 5 行：每轮减 20 血
- 第 7-8 行：但如果血量低于 100，立马回到 1000——条件永远满足，循环永远不停
- 这就是死循环——请按`Ctrl+C`强制退出

**正确的写法**——给锁血加一个计数器，不能无限回：

```python
import time
boss = {"hp": 1000, "攻击力": 50, "名字": "恶龙"}
复活次数 = 0   # 已经复活过几次了？

while boss["hp"] > 0:
    boss["hp"] -= 120
    print(f"造成 120 点伤害！恶龙剩余 hp：{boss['hp']}")
    time.sleep(1)   # 暂停 1 秒
    if boss["hp"] < 100 and 复活次数 < 3:
        boss["hp"] = 1000
        复活次数 += 1
        print(f"恶龙发动「龙血复苏」！（剩余复活次数：{3 - 复活次数}）")
        time.sleep(1)   # 暂停 1 秒

print("恶龙倒下了！！！")
```

**逐行解读**：

- 第 2 行：增加复活计数器 `复活次数`
- 第 7 行：锁血条件加了一条 `复活次数 < 3`——最多复活 3 次
- 第 9 行：每次复活计数器 +1
- 第 4 次触发时，条件不满足——终于能打死了

### while 和 for 的对比

|          | for 循环                    | while 循环                 |
| -------- | --------------------------- | -------------------------- |
| 使用场景 | 知道次数——"遍历 5 件战利品" | 不知道次数——"打到一方倒下" |
| 结束条件 | 名单遍历完自动结束          | 条件变为 False 才结束      |
| 典型写法 | `for item in list:`         | `while hp > 0:`            |
| RPG 例子 | 清点背包、给全队回血        | 战斗主循环、等待玩家输入   |
| 最大风险 | 几乎不会死循环              | 忘记更新条件 → 死循环      |

**记忆口诀**：for 有名单（list、range），while 有条件（True/False）。

### 场景二：让 AI 帮你找死循环

> "下面这段 while 循环运行后永远停不下来，帮我找出死循环的原因，只改最少代码修复它。"

```python
hp = 100
while hp > 0:
    print(f"当前 hp：{hp}")
    伤害 = 15
    # hp = hp - 伤害  ← 这行被注释掉了！永远不减血！
```

AI 会指出：循环体内没有更新 `hp` 的代码，条件 `hp > 0` 永远为 True。

**一句话总结**：`while` 是回合制战斗的心脏——"只要双方都活着，你一刀我一刀就继续"——for 需要知道终点在哪，while 只需要知道停下来的条件。
