# 「勇者斗恶龙」游戏化编程教学网站 —— 需求文档

> 版本：v1.1 | 日期：2026-04-29 | 状态：需求已确认

## 一、项目概述

### 1.1 一句话描述

一个让零基础学员在 RPG 冒险中学会 Python 编程的网页应用——在浏览器里写代码打怪物、升级、存档，17 章通关后收获一个真实可玩的文字 RPG 游戏。

### 1.2 核心价值主张

| 传统的编程课 | 这个网站 |
|------------|---------|
| 看视频被动学 | **动手写代码，即时反馈** |
| 学完不知道能干嘛 | **每章推进冒险进度，17章打完恶龙** |
| 没有成就感 | **HP条、经验值、装备、排行榜——游戏即学习** |
| 遇到问题卡住 | **内置 AI 魔法书，随时求助** |
| 一个人学孤独 | **排行榜、公会系统，看见别人也在冒险** |

### 1.3 目标用户

- **主要用户**：完全零基础的编程学习者（18-35岁）
- **使用场景**：独自在电脑前，每天 30-60 分钟，跟着课程边学边写代码
- **前置条件**：会用浏览器、打字、有自己的电脑（Windows/Mac）

### 2.3 RPG 游戏系统

#### 2.3.1 主地图

- 一张世界地图，17 个节点对应 17 章
- 从「冒险者公会」（第1章）出发，终点是「恶龙之巢」（第17章）
- 已通关的节点显示金色，当前节点闪烁，未解锁灰色
- 节点之间有路径动画（通关后路径点亮）

#### 2.3.2 角色属性与成长

```
勇者 刘老三        等级 15
══════════════════════
HP  ████████░░  320/320
MP  ██████░░░░  85/100
EXP ████████░░  2750/3000
══════════════════════
ATK 65  DEF 40
金币 3840
称号「史莱姆克星」
```

- 初始属性：HP 100, MP 50, ATK 15, DEF 5, LV 1, 金币 0
- 完成章节 → 获得经验值 + 金币
- 经验值满 → 升级（HP上限+20, ATK+3, DEF+2）
- 每章经验值：章节号 × 100（第1章100EXP，第17章1700EXP）
- 升级曲线：LV n 需要 n×200 EXP

#### 2.3.4 战斗系统

- 每章末尾有一个「Boss 战」
- Boss 战是本章知识点的综合练习（代码挑战）
- 完成 Boss 战 → 章节通关 → 获得经验+金币+掉落物品
- Boss 列表（10 个）：
  - 第1-4章：无 Boss（学习基础知识）
  - 第5章 Boss：「条件判断」哥布林队长（条件判断挑战）（注：「条件判断」为名字，"哥布林"为种族；种族无实际意义，名字直接对应本章挑战，以下类推。）
  - 第6章 Boss：「缩进」岩石巨人（for 循环挑战）
  - 第7章 Boss：「循环」骷髅将军（while 循环挑战）
  - 第9章 Boss：「函数」暗影骑士（函数挑战）
  - 第10章 Boss：「背包魔王」（列表与元组挑战）
  - 第11章 Boss：「数据结构」火元素（数据结构挑战）
  - 第12章 Boss：「字符串妖精」（字符串处理挑战）
  - 第14章 Boss：「文件操作」古代守护者（文件操作挑战）
  - 第15章 Boss：「错误处理」地狱犬（错误处理挑战）
  - 第17章 Boss：「白帝之子」恶龙（综合挑战——完整文字RPG战斗）

- Boss 战测试规范：每个 Boss 战提供 3-5 组（输入 → 期望输出），学员代码通过 stdin 读取输入、stdout 输出结果，系统比对 stdout 与期望值，全部通过即击败 Boss。

- Boss 战流程：
  1. Boss 登场，同勇者进行交互式对话（无厘头吐槽，打破第四面墙，与本章知识点强关联）。勇者用上下键选择回复 → Enter 确认
  2. 对话结束，显示 Boss 属性面板和挑战题目
  3. 学员写代码 → 运行 → 系统用预定义的多组测试用例（输入→期望输出）验证
  4. 全部测试通过 → 击败 Boss → 播放胜利动画 → 获得奖励
  5. 第17章 Boss 战胜利后，播放特殊通关画面：「两千年前，刘邦斩白蛇，提三尺剑取天下。两千年后，刘老三斩白帝之子，用 17 章 Python 征服了代码世界。你祖宗用的是剑，你用的是 print()。时代变了。」

  发出低沉的笑声：「错。因为他们太认真了。一个教编程的游戏，骷髅在那纠结 break，骑士在那纠结 return——而我，根本不在乎代码规范。」
  站起来，翅膀遮住了整个屏幕：「我用全局变量。我的函数超过 200 行。我的字典和列表不区分的混着用。我不写注释。我的编码是 GB2312。我缩进用空格和 Tab 混着来。而且你知道吗——我的代码跑得通。没人知道为什么。连我自己都不知道。」
  低头，一只眼睛对准你：「你以为这是最后一关，考的是你学过的所有东西？不。这一关考的是——看到我这种代码，你有没有勇气说：『让我重构一下』。」

  恶龙突然停住，鼻子凑近屏幕：「等等。你叫什么名字？」

  「……刘老三？？？」

  恶龙后退三步，翅膀撞塌了一座金山：「姓刘的！！两千年前，你祖宗在芒砀山砍了一条白蛇。那条蛇是我太爷爷。」

  「从那以后，我们白帝蛇族改名叫 Python。你知道为什么吗？因为『白蛇』写在简历上不好看。『Python』听起来像跨国公司。」

  「所以——你是来学编程的，还是来算家族旧账的？」

  重新站定，低头：「算了。反正你祖宗用三尺剑斩蛇，你用 keyboard 写代码。来吧，刘老三。赢了，你就是『赤帝之子』。输了——欢迎来我的代码库做维护，我们家族等了 2200 年就为了找一个姓刘的免费劳动力。」

## 三、非功能需求

### 3.1 性能

- 首屏加载（LCP）< 2 秒
- 代码编辑器启动 < 1 秒
- 代码运行响应（沙箱执行）< 3 秒
- 支持 500 并发用户

### 3.2 安全

- 后端沙箱：Docker 容器隔离，每次执行创建新容器，执行完销毁
- 代码执行限制：超时 10 秒、内存限制 256MB、禁止网络访问、禁止文件系统写入（除 /tmp）
- API 限流：同一用户每秒最多 3 次代码提交
- SQL 注入防护、XSS 防护、CSRF Token

### 3.3 兼容性

- 浏览器：Chrome 90+, Firefox 90+, Edge 90+, Safari 15+
- 屏幕：≥ 1024px 宽度（Phase 1 桌面优先，移动端适配推迟到 Phase 4）

### 3.4 可维护性

- 课程内容用 Markdown 文件管理（含 frontmatter 元数据和练习定义），支持热更新，具体 Markdown 模板格式在开发阶段定义
- 游戏数值（装备价格、经验曲线、HP 成长）通过配置文件管理，修改后 Git 提交即部署生效
