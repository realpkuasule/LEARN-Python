## 核心讲解

### 16.1 列表推导式——一键整理背包

还记得课 10 学过的列表遍历吗？每次要从背包里筛东西，都要写一个 `for` 循环，建一个空列表，然后一个个判断、一个个往里塞。逻辑没问题，但写多了就烦。

来看一个真实场景——勇者刘老三的背包，混着装备和消耗品：

```python
# 背包里有装备和消耗品——只提取装备名
backpack = ["铁剑", "回复药", "钢盾", "解毒草", "魔法杖", "魔力药"]

# 传统写法
weapons = []
for item in backpack:
    # 检查是否以「剑」「盾」「杖」结尾（课12.3 学的 endswith）
    if item.endswith("剑") or item.endswith("盾") or item.endswith("杖"):
        weapons.append(item)

print(weapons)  # → ["铁剑", "钢盾", "魔法杖"]
```

**逐行解读**：

- 第 2 行：勇者的背包，混装了 6 件物品——有武器（剑、盾、杖）也有消耗品（药水、草）。
- 第 4 行：先创建一个空列表 `weapons`，准备装筛选结果。
- 第 5 行：用 `for` 循环逐个检查背包里的每件物品。
- 第 6-7 行：用 `endswith` 判断物品名是否以「剑」「盾」「杖」结尾——三个条件用 `or` 连接，满足任意一个就算武器。不用 `any()`，直接 `or` 更直观。
- 第 8 行：符合条件的装进 `weapons` 列表。

传统写法 6 行，不算长。但要是每个筛选逻辑都这样写一遍，代码会越拉越长。列表推导式帮你把同样的逻辑压缩成一行：

```python
# 推导式——一行出活（同样的逻辑，更简洁）
weapons = [item for item in backpack
           if item.endswith("剑") or item.endswith("盾") or item.endswith("杖")]
print(weapons)  # → ["铁剑", "钢盾", "魔法杖"]
```

**逐行解读**：

- `[item for item in backpack if ...]`——读法：「对于背包里的每个 item，如果满足条件，就把它放进新列表。」
- `for item in backpack`：遍历背包（和传统写法的 for 循环一样）。
- `if item.endswith("剑") or item.endswith("盾") or item.endswith("杖")`：筛选条件——三个 `endswith` 用 `or` 连接。
- 最前面的 `item`：满足条件的元素原样放入结果列表。

**冒险类比**：你打开背包界面，点「整理」按钮——系统瞬间把武器分到武器栏、药水分到消耗品栏。推导式就是那个「整理」按钮的代码实现。

再看一个实战例子——汇总打怪经验值：

```python
# 怪物掉落经验值汇总
slimes_killed = [5, 3, 8, 2, 6]   # 每只史莱姆的经验值
total_exp = sum(slimes_killed)     # → 24
```

**逐行解读**：

- `slimes_killed` 是一个列表，记录了打死的 5 只史莱姆各自掉落的经验值。
- `sum()` 是 Python 内置函数，对列表里的数字求和——不用写 for 循环累加了。

### 16.2 字典与集合推导式——筛选图鉴

推导式不止能处理列表。字典和集合也有对应的推导式语法。

**字典推导式**——从怪物击败记录里筛出「刷怪狂人」的数据：

```python
# 怪物图鉴：{怪物名: 击败次数}
kills = {"史莱姆": 15, "哥布林": 23, "骷髅兵": 8, "恶龙": 1}

# 只显示击败超过 10 只的（刷怪狂人）
farmed = {name: count for name, count in kills.items() if count > 10}
print(farmed)  # → {"史莱姆": 15, "哥布林": 23}
```

**逐行解读**：

- `kills.items()`：把字典拆成一对一对的 `(怪物名, 击败次数)`。
- `for name, count in kills.items()`：遍历每一对，分别用 `name` 和 `count` 接收。
- `if count > 10`：只保留击败超过 10 只的。
- `name: count`：结果字典的键值对格式——冒号左边是键，右边是值。

**集合推导式**——火系法师找猎物：

```python
# 从敌人队伍里找出所有属性弱点
enemies = [
    {"name": "史莱姆", "weakness": "火"},
    {"name": "冰龙", "weakness": "火"},
    {"name": "树精", "weakness": "火"},
    {"name": "水元素", "weakness": "雷"},
]
fire_weak = {e["name"] for e in enemies if e["weakness"] == "火"}
print(fire_weak)  # → {"史莱姆", "冰龙", "树精"}  火系法师狂喜
```

**逐行解读**：

- `enemies` 是一个列表，每个元素是一个字典（怪物属性卡）。
- `{e["name"] for e in enemies if e["weakness"] == "火"}`：花括号 `{}` 表示生成集合。遍历每个敌人，只取弱点是「火」的，提取它的 `name`。
- 结果是一个集合——自动去重，格式干净。

**冒险类比**：图鉴筛选——「我火球术满级了，看看哪些怪怕火」。字典推导式一键列出，比一个一个翻图鉴快一万倍。

### 16.3 生成器与 yield——逐帧播放战斗日志

公会会长给刘老三布置了一个任务：「这是公会三百年来的战斗记录，文件大小 2GB。你帮我统计里面所有伤害的总和。」

刘老三第一反应：`f.read()` 全读进来。然后他发现——内存直接爆了。

2GB 的文件，一次全读进来，电脑根本扛不住。这时候该 `yield` 出场了：

```python
def read_battle_log(filename):
    """读取超长战斗日志——一行一行处理，不占内存"""
    import re
    with open(filename, "r") as f:
        for line in f:
            if "伤害" in line:
                yield line.strip()

# 处理十万行战斗日志，内存只占一行的大小
import re
total_damage = 0
for line in read_battle_log("epic_battle.log"):
    damage = int(re.search(r"造成(\d+)点伤害", line).group(1))
    total_damage += damage
print(f"这场战斗总共造成了 {total_damage} 点伤害！")
```

**逐行解读**：

- 第 3 行：函数内部 `import re`——生成器函数内需要正则时，在内部导入是允许的。
- 第 4-7 行：`with open` 逐行读取文件。`for line in f` 每次只读一行进内存，读完就丢弃。
- 第 6-7 行：如果这一行包含「伤害」两个字，就用 `yield` 把它「吐」出来。`yield` 不会结束函数——函数暂停在这里，等你下次再要下一行。
- 第 10 行：**函数外部也需要 `import re`**——因为外面的代码也要用 `re.search` 提取数字。
- 第 11-13 行：`for line in read_battle_log(...)`——每循环一次，生成器就「吐」一行。内存里始终只存当前这一行。
- 第 12 行：`re.search(r"造成(\d+)点伤害", line).group(1)`——从这行文字里提取伤害数字。

#### 关键区别：return vs yield

|          | `return`           | `yield`            |
| -------- | ------------------ | ------------------ |
| 返回次数 | 一次，返回整个结果 | 多次，每次返回一个 |
| 函数状态 | 执行完就结束       | 暂停，等你下次调用 |
| 内存占用 | 结果全在内存里     | 只占当前一个元素   |
| 类比     | 把整本书复印给你   | 一页一页念给你听   |

**冒险类比**：战斗回放——你不需要把整个回放一次性加载到脑子里。一帧一帧看，看完一帧忘掉前一帧。`yield` = 回放的「逐帧播放」按钮。

### 16.4 Lambda 与高阶函数——战利品结算

打完 boss，地上掉了一堆战利品。公会规定：按价值排序、高亮稀有掉落（价值 > 100）、半价回收普通物品。

这些操作有一个共同特征：**需要一个临时的排序/筛选/转换规则**。你不需要给它起名字——用完就忘。这就是 `lambda` 的用武之地。

```python
# 战利品按价值排序（金币优先）
loot = [
    {"name": "铜剑", "value": 50},
    {"name": "龙鳞", "value": 5000},
    {"name": "回复药", "value": 30},
    {"name": "传说之剑", "value": 99999},
]

# 按价值从高到低排
sorted_loot = sorted(loot, key=lambda item: item["value"], reverse=True)
print(sorted_loot)
# → 传说之剑 > 龙鳞 > 铜剑 > 回复药

# 筛出价值超过 100 的（值得捡的）
valuable = list(filter(lambda item: item["value"] > 100, loot))
print(valuable)
# → [{"name": "龙鳞", "value": 5000}, {"name": "传说之剑", "value": 99999}]

# 所有物品打折甩卖（半价）
half_price = list(map(lambda item: item["value"] // 2, loot))
print(half_price)
# → [25, 2500, 15, 49999]
```

**逐行解读**：

- `lambda item: item["value"]`：这就是一个没有名字的函数。`item` 是参数，`item["value"]` 是返回值。等价于 `def get_value(item): return item["value"]`，但不需要起名字。
- `sorted(loot, key=..., reverse=True)`：`key` 参数指定「按什么排序」——这里用 lambda 指定「按 value 字段排」。
- `filter(lambda item: item["value"] > 100, loot)`：`filter` 返回一个迭代器，只保留 lambda 返回 `True` 的元素。用 `list()` 包一层才能打印出来。
- `map(lambda item: item["value"] // 2, loot)`：`map` 对每个元素执行 lambda，返回一个新的迭代器。`// 2` 整除半价。

**什么时候用 lambda，什么时候用 def？**

| lambda                       | def                    |
| ---------------------------- | ---------------------- |
| 临时用一次，规则极简单       | 需要复用，逻辑超过一行 |
| 传给 `sorted`/`filter`/`map` | 有名字更好理解         |
| 用完就忘，不污染命名空间     | 多个地方调用同一个逻辑 |

**冒险类比**：战利品结算界面。你一点「结算」——物品自动按价值排序（`sorted`），稀有掉落高亮闪烁（`filter`），商店回收价自动算好（`map`）。lambda 就是结算界面每条结算规则的「临时便利贴」——用完就撕，不占地方。
