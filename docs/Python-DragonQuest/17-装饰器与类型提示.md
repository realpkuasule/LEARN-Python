## 核心讲解

### 17.1 装饰器是什么——给技能套上自动记录

回想一下：你在课 8 写的每一个技能函数（`attack`、`cast_spell`、`use_potion`），它们只负责「执行技能效果」。但公会需要一个额外的功能：**每次技能发动，自动记录时间和名称。**

你当然可以在每个技能函数里面加一行 `print(f"[日志] {技能名} 发动！")`。但如果有 20 个技能呢？改 20 个地方？今天要加日志，明天要加计时，后天要加伤害统计——每次都改 20 个函数，迟早改漏。

装饰器解决的就是这个问题：**不改原函数，给它套一层「外壳」。**

```python
import time
import random

def combat_log(func):
    """装饰器——自动记录每次技能使用的效果"""
    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)   # 执行原技能
        elapsed = time.time() - start
        print(f"[战斗日志] {func.__name__} 发动！耗时 {elapsed:.2f}s")
        return result
    return wrapper

@combat_log
def dragon_breath(caster, targets):
    """恶龙的龙息——造成全队 AOE 伤害"""
    for target in targets:
        damage = random.randint(80, 120)
        target["hp"] -= damage
    return True

# 每次龙息发动——自动记录时间！不用改龙息函数本身
boss = {"name": "恶龙", "hp": 500}
party = [{"name": "勇者", "hp": 100}, {"name": "法师", "hp": 60}, {"name": "牧师", "hp": 80}]
dragon_breath(boss, party)
# 输出 → [战斗日志] dragon_breath 发动！耗时 0.05s
```

**逐行解读**：

- 第 1-2 行：导入 `time`（计时）和 `random`（随机伤害）。
- 第 4 行：`combat_log(func)`——装饰器本身是一个函数，参数 `func` 是被装饰的原始函数。
- 第 6 行：`wrapper(*args, **kwargs)`——这是「外壳」函数。`*args` 和 `**kwargs` 意味着它可以接收原函数的任意参数，照原样传进去。
- 第 7 行：`start = time.time()`——记录技能发动前的时间戳。
- 第 8 行：`func(*args, **kwargs)`——执行原始技能函数，拿到返回值。
- 第 9 行：计算技能耗时。
- 第 10 行：打印战斗日志——`func.__name__` 是原函数的名字（如 `"dragon_breath"`）。
- 第 11 行：返回原函数的返回值——这样调用者拿到的结果和没有装饰器时完全一样。
- 第 14 行：`@combat_log`——这就是装饰器的语法糖。等价于 `dragon_breath = combat_log(dragon_breath)`。
- 第 23-24 行：**先定义 `boss` 和 `party`，再调用 `dragon_breath`**——变量必须在使用前定义。
- 第 25 行：调用龙息——自动触发战斗日志输出。

**执行流程——装饰器到底干了什么**：

```markdown
dragon_breath(boss, party)
│
▼
wrapper(boss, party) # 装饰器把 dragon_breath 替换成了 wrapper
│
├── start = time.time() # 记开始时间
├── dragon_breath(boss, party) # 执行原函数（龙息效果）
├── print(战斗日志) # 打印日志
└── return result # 返回原函数的结果
```

**冒险类比**：游戏内置的战斗记录器。你不用手动记「第几回合谁用了什么技能」——系统自动帮你记。装饰器 = 给你的技能函数套了一层「自动记录」的外壳。技能本身该怎么写还怎么写，记录的事交给外壳。

### 17.2 常见内置装饰器——@property 装备面板

Python 自带了一些装饰器，最常用的是 `@property`。它让方法看起来像属性——不用加括号调用。

在 RPG 里最经典的场景：装备面板的攻击力显示。

```python
class Weapon:
    def __init__(self, name, base_atk, enchant=0):
        self.name = name
        self._base_atk = base_atk
        self._enchant = enchant

    @property
    def atk(self):
        """攻击力 = 基础攻击 + 附魔加成"""
        return self._base_atk + self._enchant

sword = Weapon("传说之剑", 80, enchant=15)
print(sword.atk)   # → 95（直接读 sword.atk，不用 sword.atk()！）
print(sword.name)  # → "传说之剑"（普通属性）
```

**逐行解读**：

- 第 5-6 行：`_base_atk` 和 `_enchant`——下划线开头表示「这是内部数据，别直接从外面改」。
- 第 8-10 行：`@property` 把 `atk` 方法变成了属性。外面读 `sword.atk` 时，实际上悄悄执行了这一段计算。
- 第 13 行：`sword.atk`——不需要加括号！但背后自动算了 `80 + 15 = 95`。
- 如果以后加了 Buff 系统（比如「攻击力翻倍」），只需在 `atk` 方法里改计算逻辑，所有读 `sword.atk` 的地方自动更新。

**冒险类比**：装备属性面板。你看到的攻击力永远是「基础 + 附魔 + Buff」的最终值。`@property` = 面板自动帮你算好了，不需要你每次手动加。加一行附魔、加一个 Buff，面板数字自动变。

### 17.3 类型提示——给变量挂上「属性铭牌」

你还记得课 3 学过变量是「给数据贴标签」吗？类型提示就是给标签再挂一个「类型铭牌」——标清楚这到底是整数、小数、字符串还是布尔值。

```python
# 角色创建——标清楚每个属性是什么类型
hero_name: str = "勇者刘老三"
hero_level: int = 1
hero_hp: int = 100            # HP 是整数——没有半滴血
hero_crit_rate: float = 0.15  # 但暴击率可以是小数
is_chosen_one: bool = True

def attack(attacker: dict, defender: dict, skill_name: str = "普通攻击") -> int:
    """攻击！返回造成的伤害值"""
    damage = attacker["atk"] - defender.get("def", 0)
    if damage < 1:
        damage = 1
    defender["hp"] -= damage
    return damage
```

**逐行解读**：

- `hero_hp: int = 100`——类型提示是 `: int`，表示「这个变量应该是整数」。HP 没有半滴，所以是 `int`。
- `hero_crit_rate: float = 0.15`——暴击率 15%，是小数，所以用 `float`。
- `is_chosen_one: bool = True`——天命之人？True 或 False，用 `bool`。
- 第 7 行：`def attack(attacker: dict, defender: dict, skill_name: str = "普通攻击") -> int:`——参数标注类型，箭头 `-> int` 标注返回值类型（伤害值是整数）。

**类型提示不强制类型**：Python 不会因为你写了 `: int` 就阻止你赋值 `hero_hp = "一百"`。类型提示是「铭牌」——挂在变量旁边，让人和 AI 看清楚这个变量的预期类型。编辑器会给出黄色波浪线提示，但代码照样能跑（直到运行时才可能出错）。

**冒险类比**：角色创建界面的属性字段——姓名是文字（`str`）、等级是整数（`int`）、HP 是整数（`int`）、暴击率是小数（`float`）。类型提示 = 属性栏旁边的标签——告诉你（和 AI）这个字段该填什么类型的值。

**AI 场景——类型提示让 AI 秒懂你的代码**：你写 `def heal(target: dict, amount: int) -> int:`——AI 一眼就懂：`target` 是角色字典，`amount` 是整数回血量，返回实际回血值。比你写三行注释描述参数还快。你写 `def find_monster(name: str) -> Optional[Monster]:`——AI 秒懂：可能返回 Monster 对象，也可能找不到返回 None。这比「说说你的需求让 AI 猜」高效十倍。

### 17.4 @dataclass——怪物图鉴卡片模板

写 RPG 游戏，你可能需要定义很多「数据类」——怪物、装备、药水。每个类都要手写 `__init__`，还要写 `__repr__` 才能在 `print` 时显示好看的信息。很烦。

`@dataclass` 装饰器帮你自动生成这些样板代码：

```python
from typing import Optional
from dataclasses import dataclass

@dataclass
class Monster:
    name: str
    hp: int
    atk: int
    defense: int = 0
    element: str = "无"
    is_boss: bool = False

# 自动生成 __init__、__repr__、__eq__
slime = Monster(name="史莱姆", hp=30, atk=8)
boss = Monster(name="恶龙", hp=5000, atk=120, defense=50, element="火", is_boss=True)

print(slime)
# → Monster(name='史莱姆', hp=30, atk=8, defense=0, element='无', is_boss=False)
print(boss)
# → Monster(name='恶龙', hp=5000, atk=120, defense=50, element='火', is_boss=True)

def find_monster(name: str) -> Optional[Monster]:
    """查怪物图鉴——可能还没遇到，返回 None"""
    return monster_handbook.get(name)
```

**逐行解读**：

- 第 1 行：从 `typing` 模块只导入 `Optional`——不需要 `Union`，`Optional[X]` 就等价于 `Union[X, None]`，写起来更简洁。
- 第 2 行：`@dataclass` 是 `dataclasses` 模块提供的装饰器，独立于 `typing`。
- 第 4-10 行：`Monster` 类——每一行是一个带类型提示的字段。`defense: int = 0` 表示有默认值 0；没有默认值的字段（`name`、`hp`、`atk`）必须在创建时提供。
- 第 13 行：创建史莱姆——只给了三个必填字段，其他用默认值。
- 第 14 行：创建恶龙——所有字段都指定了值。
- 第 16-18 行：`print(slime)` 自动输出漂亮的格式——`@dataclass` 自动生成了 `__repr__`。
- 第 21-23 行：`find_monster` 返回类型是 `Optional[Monster]`——意思是「要么返回一个 Monster 对象，要么返回 None」。

**@dataclass vs 手写类**：

|            | 手写                                  | @dataclass                     |
| ---------- | ------------------------------------- | ------------------------------ |
| `__init__` | 手写，每个字段写一行 `self.xxx = xxx` | 自动生成                       |
| `__repr__` | 手写或看不见                          | 自动生成可读格式               |
| `__eq__`   | 手写或默认比较内存地址                | 自动按字段值比较               |
| 类型提示   | 可加可不加                            | 强制加（字段定义就是类型提示） |

**冒险类比**：怪物图鉴卡片模板——每张卡格式一样（名字、HP、攻击、防御、属性、是否BOSS），但数值不同。`@dataclass` = 那个卡片印刷模板——你填数据就行，格式自动统一。新加一种怪物？一行代码，卡片印好。

## AI 协作指南

### 场景一：让 AI 帮你写装饰器

装饰器对新手来说，嵌套函数的写法确实绕。最有效的方式：描述你想要的「附加功能」，让 AI 帮你生成外壳代码。

> **Prompt 示例**：「帮我写一个 Python 装饰器 `@skill_cooldown`，用于 RPG 技能系统。它的功能是：被装饰的技能函数，如果距离上次使用不满 N 秒（冷却时间），就打印『技能冷却中，还需 X 秒』并跳过执行；冷却好了就正常执行。技能冷却时间可以从函数名推断，或通过装饰器参数指定（比如 `@skill_cooldown(3)` 表示冷却 3 秒）。写出完整代码和用法示例。」

你会看到 AI 如何构造装饰器嵌套结构——外层接收参数，内层接收函数，最内层 wrapper 执行逻辑。读一遍，你就懂了装饰器的参数传递。

### 场景二：用类型提示让 AI 理解你的代码结构

类型提示最大的受益者不是 Python 解释器——是 AI 和你的人类队友。试试这个对比实验：

> **Prompt 示例（不带类型提示）**：「帮我写一个 Python 函数 battle_result，接收 hero 和 monster 两个参数，返回战斗结果。」
> **Prompt 示例（带类型提示）**：「帮我写一个 Python 函数 `battle_result(hero: dict, monster: Monster) -> str`，其中 `Monster` 是一个 `@dataclass`，字段有 `name: str, hp: int, atk: int`。函数比较双方战力，返回 `"victory"`、`"defeat"` 或 `"escaped"`。」

两个 Prompt 发给 AI——你会发现第二个得到的代码精准得多。类型提示就是你和 AI 之间的「精准战术指令」——你不说清楚谁是步兵谁是弓兵，AI 就会让弓兵去肉搏。

## 实战练习

### 练习1：给技能加日志装饰器（简单）

写一个装饰器 `@log_skill`，让任何被它装饰的技能函数在执行时打印 `[技能日志] {技能名} 已发动！`。然后把它应用到下面两个函数上：

```python
@log_skill
def slash(target):
    target["hp"] -= 20

@log_skill
def heal(target):
    target["hp"] += 30
```

测试：创建 `target = {"hp": 50}`，依次调用 `slash(target)` 和 `heal(target)`，观察日志输出。

### 练习2：怪物图鉴系统（中等）

用 `@dataclass` 创建一个 `Monster` 数据类，字段包括 `name: str`、`hp: int`、`atk: int`、`defense: int = 0`、`element: str = "无"`。

然后完成：

1. 创建 3 只怪物：史莱姆（hp=30, atk=8）、哥布林（hp=50, atk=12）、恶龙（hp=5000, atk=120, defense=50, element="火"）。
2. 写一个函数 `find_weakness(monster_list: list[Monster], element: str) -> list[str]`，带完整类型提示，返回所有弱点是 `element` 的怪物名字列表。这里需要给 `Monster` 加一个 `weakness: str = ""` 字段。
3. 用 `Optional` 提示写一个查找函数 `lookup(name: str, handbook: dict) -> Optional[Monster]`。

### 练习3：战斗日志自动记录系统（综合）

整合本章所有知识点，创建一个完整的战斗日志系统：

1. 写一个带参数的装饰器 `@timed(label)`，接受一个标签参数（如 `"勇者攻击"`、`"龙息"`），执行技能时打印 `[{label}] 发动！耗时 {秒数}s`。
2. 用 `@dataclass` 创建 `BattleLog` 类（字段：`action: str`、`damage: int`、`timestamp: float`），用来存储每一条日志。
3. 写一个带完整类型提示的函数 `analyze_logs(logs: list[BattleLog]) -> dict[str, int]`，统计每个 action 的累计伤害，返回 `{"勇者攻击": 150, "龙息": 320}`。
