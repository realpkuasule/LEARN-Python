## 核心讲解

### 9.1 return 的用法——战斗函数必须回报战果

在 RPG 里，每次战斗结束后你都需要知道结果：打赢了获得经验和金币，团灭了回城镇复活，逃跑了则什么都不发生。函数里的 `return` 就是做这件事的——它把函数内部的结果「交回」给调用者。

```python
def battle(hero, monster):
    """战斗（第7课的循环封装成了函数）！返回结果"""
    while hero["hp"] > 0 and monster["hp"] > 0:
        # 战斗逻辑（和课7一样的回合制循环）
        action = input("1.攻击 2.魔法 3.道具 4.逃跑 > ")
        if action == "4":
            return "escaped"   # 跑了——战斗结束，返回结果
        # ...攻击和反击逻辑...

    if hero["hp"] <= 0:
        return "defeat"        # 团灭
    else:
        return "victory"       # 胜利！

# 根据战斗结果做不同的事
hero = {"name": "勇者刘老三", "hp": 100, "atk": 15, "def": 5}
dragon = {"name": "恶龙", "hp": 500, "atk": 80, "def": 30}
result = battle(hero, dragon)
if result == "victory":
    print("击败了恶龙！")
    # level_up(hero) ← 后面可以加上升级逻辑
    # open_treasure_chest() ← 后面可以加上宝箱系统
elif result == "defeat":
    print("勇者倒下了...Game Over")
elif result == "escaped":
    print("勇者逃回了城镇...")
```

**逐行解读：**

- `def battle(hero, monster):` —— 定义一个战斗函数，接收勇者和怪物两个参数。
- `while hero["hp"] > 0 and monster["hp"] > 0:` —— 战斗主循环，只要双方都活着就继续。
- `if action == "4": return "escaped"` —— 玩家选逃跑，函数立刻结束，返回字符串 `"escaped"`。`return` 后的代码不会再执行。
- `if hero["hp"] <= 0: return "defeat"` —— 循环结束，检查勇者血量。如果勇者倒下了，返回 `"defeat"`。
- `else: return "victory"` —— 否则就是怪物倒了，返回 `"victory"`。
- `hero = {...}` 和 `dragon = {...}` —— **在调用 battle 前先定义好勇者和恶龙的属性卡。**
- `result = battle(hero, dragon)` —— 调用战斗函数，把返回值存到变量 `result` 里。
- `if result == "victory":` —— 根据返回值做不同的后续处理。这就是 `return` 的价值：调用者拿到结果后可以决定下一步做什么。

**类比**：你去冒险者公会交任务——打了一只怪物，回来报告结果：「打赢了」「团灭了」「逃跑了」。`return` 就是你回来交差时说的那句话。没有 `return`，公会前台永远不知道你到底是死是活。

### 9.2 变量的作用域——队伍频道 vs 世界频道

函数内部定义的变量只在函数内部有效，外面访问不到。这就是「作用域」。

```python
def boss_fight():
    boss_secret = "恶龙的弱点是尾巴尖"   # 只在 boss 战里知道
    return boss_secret

# print(boss_secret)   ← 报错！战斗中发现的秘密，出了战斗就没人知道

world_name = "阿尔德大陆"   # 全局——所有人都知道这个世界叫什么
```

**逐行解读：**

- `boss_secret = "恶龙的弱点是尾巴尖"` —— 这个变量定义在函数内部，是局部变量。
- `return boss_secret` —— 如果想把这个秘密传出去，必须用 `return`。
- `# print(boss_secret)` —— 注释掉说明这会报错。函数外的代码看不到 `boss_secret`。
- `world_name = "阿尔德大陆"` —— 在所有函数外面定义，是全局变量，任何地方都能访问。

**类比**：队伍频道 vs 世界频道——队伍里讨论的战术（局部变量），敌人听不到。但世界公告「魔王已死」（全局变量），所有 NPC 都知道了。

**好的设计原则**：用参数传情报，用 `return` 回报战果——不走「世界频道广播」。全局变量越少越好，因为它们可以被任何函数修改，出了 bug 很难排查。

### 9.3 什么时候该拆函数——给代码分系统

一个函数既选指令、又算伤害、还播动画、顺带记日志——100 行代码揉在一起，改一行可能崩三处。这时候需要拆分。

```python
# 拆之前——「战斗」一个函数干了四件事
def battle(...):
    # 选指令 → 算伤害 → 播动画 → 记录日志
    # 100 行，改起来头疼

# 拆之后——每件事一个函数
def select_action(hero): ...        # 选指令
def calculate_damage(atk, def_): ... # 算伤害
def play_animation(action): ...      # 播动画
def log_battle(turn, event): ...    # 记日志
```

**逐行解读：**

- 拆之前，`battle` 函数包揽一切——选指令、算伤害、播动画、记日志全挤在一起。
- 拆之后，每个职责一个独立函数——`select_action` 负责选指令，`calculate_damage` 负责算伤害，`play_animation` 负责播动画，`log_battle` 负责记日志。
- 注意 `def_` 的参数名——因为 `def` 是 Python 的关键字不能用作变量名，所以加了下划线。

**三条铁律**：

1. 超过 20 行 → 拆。一个函数太长就说明它干了太多事。
2. 重复两次 → 拆。同样的逻辑写了两遍？封装成函数。
3. 能起个技能名 → 拆。如果你能给一段代码起个像「火焰斩」「治疗术」这样的名字，它就值得成为一个函数。

**类比**：RPG 的系统架构——战斗系统、装备系统、技能系统、存档系统——分开开发，各管各的。不会把存档逻辑写在战斗代码里。拆函数就是给代码分系统。

## 实战练习

### 练习 1：战斗结算系统

写一个 `battle_result(hero, monster)` 函数，战斗简化成直接比较 atk 属性：

- 勇者攻击力 > 怪物攻击力 × 2 → 返回 `"victory"`（碾压胜利）
- 勇者攻击力 > 怪物攻击力 → 返回 `"victory"`（普通胜利）
- 勇者攻击力 == 怪物攻击力 → 返回 `"escaped"`（势均力敌，撤退）
- 否则返回 `"defeat"`

调用前先定义 hero 和 monster 字典，接收返回值并打印不同的结算信息。

### 练习 2：作用域实验

写一段代码，包含：

1. 在全局作用域定义一个变量 `guild_name = "冒险者公会"`
2. 在函数 `complete_quest()` 内部定义一个局部变量 `reward = 500`
3. 在函数内部打印 `guild_name`（证明函数内可以访问全局变量）
4. 在函数内部打印 `reward`
5. 在函数外面尝试打印 `reward`，用 try/except 捕获 NameError 并打印「函数内的变量外面看不到！」

### 练习 3：拆分一个多功能函数

下面是一个做了太多事的函数，请把它拆分成至少 3 个独立函数：

```python
def quest_system(hero):
    # 1. 显示可用任务列表
    quests = ["讨伐史莱姆", "收集药草", "护送商队"]
    for i, q in enumerate(quests):
        print(f"[{i+1}] {q}")

    # 2. 接收玩家选择
    choice = int(input("接哪个任务？"))
    selected = quests[choice - 1]

    # 3. 检查等级要求并结算
    if selected == "讨伐史莱姆" and hero["level"] >= 1:
        hero["gold"] += 100
        return f"完成！获得 100 金币"
    elif selected == "收集药草" and hero["level"] >= 3:
        hero["gold"] += 200
        return f"完成！获得 200 金币"
    # ...
```

拆分要求：`show_quests()` 显示任务、`select_quest()` 接收选择、`complete_quest(hero, quest)` 检查条件并结算。用 `return` 传递数据，不依赖全局变量。
