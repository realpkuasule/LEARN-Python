# 第一课：编程为什么重要

---

## 冒险引入

你站在冒险者公会的石砌大厅里，壁炉的火光照得墙上的告示忽明忽暗。告示上只有一行大字：

> 「招募勇者讨伐恶龙。需自行组建队伍。不会编程者恕不接待。」

你皱起眉头，找到公会会长——一个头发花白但眼神锐利的老冒险者。「会长，」你问，「为什么打龙还要会编程？」

会长没有直接回答，而是指了指角落里一个垂头丧气的老勇者。那人面前摆着一本发光的魔法书（AI），他正对着魔法书大喊：「帮我把攻击力提上来！」魔法书嗡嗡作响，然后——把他全身装备全变成了攻击力 +1 的木剑。老勇者崩溃地瘫倒在椅子上。

会长又指了指另一个年轻的勇者。那人冷静地对魔法书说：「读取我的装备列表，把所有防御力低于 50 的装备替换成防御力 80 以上的，优先替换头盔和铠甲，保留武器不变。」魔法书精准执行，五分钟就搞定了。年轻勇者背上传说之剑，踏出了公会大门。

「看到了吗？」会长转过身看着你，「魔法书不是问题的答案——它是对你表达能力的放大器。你说得模糊，它做得离谱。你说得精准，它做得完美。」

会长忽然眯起眼：「说起来……你姓刘？叫刘老三？？上一个坐在这张椅子上的刘老三，后来不当亭长跑去造反了——对，就是刘邦。他说什么『大丈夫当如是』。然后他用一柄三尺剑，斩了一条白蛇——那条蛇叫白帝之子。他不学编程，因为他那时候还没有计算机。但是你有。」

会长拍了拍你的肩膀：「你祖宗斩的是真蛇。你要斩的是一条叫 Python 的大蟒蛇。不一样的是——你祖宗杀完之后就完事了，你要先学会怎么 import 它。奇怪吧？这就是时代进步。」

会长又指了指墙上挂着的一幅画像——一条巨大的蟒蛇，有两颗牙。「咱们这个世界的守护神兽就是 Python。」他说，「它有两颗牙——一颗叫 Python 2，一颗叫 Python 3。Python 2 那颗牙在 2020 年掉了。所以现在只剩 Python 3。等你学成了——你就会明白为什么掉的那颗牙没人怀念。」

## 学习目标

学完本章你能：

- 理解为什么 AI 时代依然需要学编程，而不是「让 AI 帮我写就行」
- 说清楚「懂编程的人用 AI」和「不懂编程的人用 AI」的本质区别
- 知道 Python 是什么、为什么选它入门，而不是 JavaScript/Go/Java
- 建立「学编程是为了打恶龙，不是为了学语法」的心态

---

## 核心讲解

### Python 是什么、为什么选它

Python 是所有编程语言里长得最像咒语的——读起来接近英文。我们用它写冒险游戏，因为写起来最省力。

**你记住：JavaScript 里 `let hp = 100` 就是 Python 里 `hp = 100`。Go 里 `var hp int = 100` 也是。换了个写法，说的都是「勇者有 100 点血」。**

我们学 Python，纯粹是因为说它最省嘴。不是为了学 Python 而学 Python——是为了打恶龙。

### AI 时代为什么还要学编程

你不需要成为「写代码最快的人」。但你必须成为「能把战术说清楚的人」。魔法书（AI）是你的放大器：

| 你不懂编程             | 你懂编程                                                         |
| ---------------------- | ---------------------------------------------------------------- |
| 「帮我把装备弄好点」   | 「把防御低于 50 的换成 80+，武器不动」                           |
| 魔法书给你全换木剑     | 魔法书精准替换，五分钟搞定                                       |
| 打龙团灭，不知道为什么 | 看战斗日志：「第 15 回合，HP 为 0 时还在攻击——没有判断死亡状态」 |

懂编程，意味着你能：

1. **精准描述问题**——AI 需要你画好边界，边界越清楚，输出越正确。
2. **读懂 AI 的答案**——AI 给了代码，你能看懂它在干什么，而不是盲目复制。
3. **检查逻辑漏洞**——AI 会说「这样就行」，但你一眼看出「不对，忘了保底 1 点伤害」。

你不是在和 AI 比赛写代码。你是导演，AI 是场务——导演说得清，场务不跑偏。

## AI 协作指南

### 场景一：让 AI 帮你理解「编程是什么」

当你面对一堆术语不知所措时，让魔法书（AI）用你的世界来解释：

> **Prompt 示例**：「我是一个刚学编程的人，正在做一个文字 RPG 游戏。请用「勇者打史莱姆」的比喻告诉我：变量、条件判断、循环分别是游戏里的什么？」

AI 会用你熟悉的冒险场景来解释概念，而不是扔一堆教科书定义。

### 场景二：用 AI 验证你的理解

学完一个概念后，你对 AI 说出你的理解，让它纠正：

> **Prompt 示例**：「我用我自己的话解释一下 Python 的变量，你帮我看看对不对——变量就是给数据贴标签，比如 `hp = 100` 就是把标签 'hp' 贴到数字 100 上。受伤了 `hp = hp - 15` 就是把标签撕下来贴到 85 上。我理解得对吗？」

这比你闷头看教科书高效得多——AI 针对你的理解水平给出反馈。

## 实战练习

### 练习1：改一句话，看 AI 表现（最简单）

打开任意一个 AI 工具（ChatGPT、Claude 等），分别输入下面两句话，观察 AI 回答的差异：

- 模糊版：「帮我写一个战斗系统」
- 精准版：「帮我用 Python 写一个文字 RPG 战斗系统：勇者攻击力 15，怪物防御力 5，伤害 = 攻击力 - 防御力，保底 1 点伤害。双方轮流攻击，直到一方 HP 归零。初始 HP：勇者 100，怪物 50。」

对比两个回答的质量、长度、可执行性，记录你的发现。

### 练习2：用「解释清楚」的方式提一个需求（中等）

你正在设计一个「背包系统」，但你还不会写代码。用文字精准描述这个需求，确保另一个懂编程的人（或 AI）能直接实现：

- 背包最多 20 格
- 每格可以放一件物品（物品有名字、类型、数量）
- 支持的操作：放入、取出、查看第 N 格、查看全部
- 有些物品可堆叠（比如回复药最多 99 个一格），有些不可堆叠（武器每件占一格）

把这段描述发给 AI，让它写 Python 代码。阅读 AI 输出的代码——你现在不需要完全看懂，但试着找出你觉得「说得对」和「可能有 bug」的地方。

### 练习3：结合 AI，规划你的冒险旅程（综合）

你即将学习 17 课内容，每课对应一个 RPG 功能。让 AI 帮你梳理：

> **Prompt 示例**：「我正在学 Python，课程用 RPG 游戏贯穿：变量（角色属性）、条件判断（战斗逻辑）、循环（回合制战斗）、函数（技能系统）、列表（背包）、字典（装备栏）、文件操作（存档）。请帮我画一个系统架构图（用文字描述），说明每个系统之间的关系和数据流向。比如：『战斗系统调用技能系统（函数），技能系统读取装备栏（字典）的武器加成，战斗结果更新角色属性（变量），最终写入存档（文件）。』」

## 本章回顾

**一句话总结**：编程不是和 AI 抢饭碗，而是让你学会用 AI 能理解的方式说清楚你的战术——你当导演，AI 当场务。
