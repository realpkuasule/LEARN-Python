## 核心讲解

### 11.1 字典基础——打开角色面板

字典是 Python 中最强大的数据结构之一。它用「键-值对」存储数据，你通过键（名字）来找值（数据）。在 RPG 里，它就是你的角色属性面板——按 C 键打开，每个属性都有名字。

```python
# 勇者的完整属性面板
hero = {
    "name": "勇者刘老三",
    "level": 15,
    "hp": 320,
    "max_hp": 320,
    "mp": 85,
    "max_mp": 100,
    "atk": 65,
    "def": 40,
    "gold": 3840,
    "exp": 2750,
}

hero["hp"]       # → 320（当前血量）
hero["atk"]      # → 65（攻击力）
hero.get("luck", 0)   # → 0（没有幸运值，默认 0）

# 获得新属性！
hero["title"] = "史莱姆克星"   # 随时加字段（终章会升级为「恶龙杀手」）
```

**逐行解读：**

- `hero = { ... }` —— 用花括号创建字典。每个键值对用冒号连接，键值对之间用逗号分隔。
- `"name": "勇者刘老三"` —— 键 `"name"` 对应值 `"勇者刘老三"`。字符串键用引号包裹。
- `"hp": 320` —— 键 `"hp"` 对应整数 320。注意使用小写简写 `hp`、`mp`、`atk`、`def`，保持统一风格。
- `hero["hp"]` —— 用方括号通过键访问值。`hero["hp"]` 返回 320。
- `hero["atk"]` —— 同样方式获取攻击力 65。注意用的是 `atk` 不是 `attack`，全课程统一。
- `hero.get("luck", 0)` —— `get()` 方法安全取值。如果键不存在，返回默认值 0 而不是报错。
- `hero["title"] = "史莱姆克星"` —— 字典是可变的，随时可以加新键值对。`"title"` 键原本不存在，这行代码会创建它。附注：终章打败恶龙后这个称号会升级为「恶龙杀手」。

**类比**：角色状态面板——你按 C 键打开，看到所有属性整齐排列。`hero["hp"]` 就是「看一眼血量」。字典就是角色面板的代码形态。你不需要记住「HP 是第 3 个属性」——只要叫出它的名字。

### 11.2 字典的常用操作——装备栏管理

字典不仅用于角色面板，装备系统也是字典的典型应用场景——每个装备槽位是一个键，对应的装备是值。

```python
# 装备栏——每个部位一件装备
equipment = {
    "武器": "传说之剑",
    "头盔": "龙鳞盔",
    "铠甲": "秘银甲",
    "饰品": "勇者之证",
}

# 查看所有装备
for slot, item in equipment.items():
    print(f"{slot}：{item}")

# 换装备
equipment["武器"] = "勇者之剑·真"   # 升级了！

# 检查有没有装备头盔
if "头盔" in equipment:
    print(f"头盔：{equipment['头盔']}")
```

**逐行解读：**

- `equipment = { "武器": "传说之剑", ... }` —— 装备字典，键是槽位名，值是装备名。
- `for slot, item in equipment.items():` —— `.items()` 方法返回所有键值对，一次取出槽位名和装备名。
- `print(f"{slot}：{item}")` —— 格式化打印每个槽位的信息。
- `equipment["武器"] = "勇者之剑·真"` —— 通过键修改值。换武器就是给「武器」这个键赋新值。
- `if "头盔" in equipment:` —— `in` 运算符检查字典中是否存在某个键。检查是否装备了头盔。

**类比**：装备界面——武器槽、头盔槽、铠甲槽、饰品槽。每个槽位（key）装一件装备（value）。换武器就是给「武器」这个 key 换一个 value。检查有没有头盔就是看看「头盔」这个 key 在不在字典里。

**常用字典方法速查**：

| 方法                     | 作用                 | 示例                           |
| ------------------------ | -------------------- | ------------------------------ |
| `dict[key]`              | 取值（键不存在报错） | `hero["hp"]`                   |
| `dict.get(key, default)` | 安全取值             | `hero.get("luck", 0)`          |
| `dict[key] = value`      | 设值/添加            | `hero["title"] = "史莱姆克星"` |
| `key in dict`            | 检查键是否存在       | `"头盔" in equipment`          |
| `dict.items()`           | 遍历所有键值对       | `for k, v in d.items():`       |
| `dict.keys()`            | 获取所有键           | `hero.keys()`                  |
| `dict.values()`          | 获取所有值           | `hero.values()`                |

### 11.3 集合——不重复的无序容器

集合是一种特殊的容器：元素不重复、无序。在 RPG 里最典型的应用是怪物图鉴——不管你遇到史莱姆多少次，图鉴里只记录一次。

```python
# 怪物图鉴——遇到了哪些怪物（自动去重）
encountered = {"史莱姆", "哥布林", "史莱姆", "骷髅兵", "哥布林"}
# → {"史莱姆", "哥布林", "骷髅兵"}  自动去重！

# 所有怪物 vs 已击败的 → 还有哪些没打过？
all_monsters = {"史莱姆", "哥布林", "骷髅兵", "恶龙", "魔王"}
defeated = {"史莱姆", "哥布林", "骷髅兵"}
remaining = all_monsters - defeated   # → {"恶龙", "魔王"}  还要打！

# 两个冒险者的图鉴合并
hero_a_seen = {"史莱姆", "哥布林"}
hero_b_seen = {"骷髅兵", "恶龙"}
combined = hero_a_seen | hero_b_seen   # → {"史莱姆", "哥布林", "骷髅兵", "恶龙"}
```

**逐行解读：**

- `encountered = {"史莱姆", "哥布林", "史莱姆", "骷髅兵", "哥布林"}` —— 用花括号创建集合（注意没有冒号，和字典的区别）。重复的元素自动被去重。
- `# → {"史莱姆", "哥布林", "骷髅兵"}` —— 集合只保留了不重复的元素，史莱姆和哥布林虽然遇到了多次，但只记录一次。
- `all_monsters = {...}` —— 全怪物列表，5 种怪物。
- `defeated = {...}` —— 已击败的怪物，3 种。
- `remaining = all_monsters - defeated` —— 差集运算。`-` 表示「在 A 中但不在 B 中」。剩下恶龙和魔王没打。
- `hero_a_seen | hero_b_seen` —— 并集运算。`|` 表示「两个集合中所有的元素」。两个冒险者交换图鉴后合并。

**类比**：怪物图鉴——你遇到史莱姆 10 次，图鉴只记录一次。差集 = 图鉴上还空着的格（没遇到的怪物）。并集 = 两个冒险者交换图鉴后合在一起。集合就是自带去重功能的图鉴系统。

## AI 协作指南

### 场景 1：让 AI 帮你设计角色属性系统

你要创建一个完整的角色系统，属性很多但又不想全部手动写：

> 请帮我设计一个勇者角色的字典结构，包含以下分类：
>
> 1. 基础属性：name、level、hp、max_hp、mp、max_mp、atk、def
> 2. 经济属性：gold、exp
> 3. 装备属性：weapon、helmet、armor、accessory
> 4. 称号属性：title
>
> 写一个 `create_hero(name, base_hp, base_mp, base_atk, base_def)` 函数，接收名字和初始属性，返回一个完整的 hero 字典。用 `get()` 方法安全访问可能不存在的属性。

### 场景 2：让 AI 帮你设计图鉴系统

你想用集合做一个完整的怪物图鉴系统：

> 请帮我写一个怪物图鉴模块：
>
> 1. 用集合 `encountered` 记录所有遇到过的怪物
> 2. 用集合 `defeated` 记录所有击败过的怪物
> 3. 写函数 `meet_monster(name)` —— 将怪物加入 `encountered`
> 4. 写函数 `defeat_monster(name)` —— 将怪物加入 `defeated`
> 5. 写函数 `get_progress()` —— 返回 (遇到总数, 击败总数, 击败率)
> 6. 写函数 `get_missing(all_monsters)` —— 接收全怪物集合，返回还没遇到过的怪物
>
> 所有函数通过参数接收集合数据，不依赖全局变量。
