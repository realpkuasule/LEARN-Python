## 核心讲解

### 安装 Python——搭建你的铁匠铺

去 [python.org](https://python.org) 下载最新版 Python。安装过程中有一件**绝对不能忘**的事：

> 勾上「Add Python to PATH」

**冒险类比**：你在冒险者公会登记姓名。登记完，城里 NPC 才能喊出你的名字、给你任务。不登记？每次进城卫兵都问你是谁。`PATH` 就是城里的居民名册——把你加进去，全城的 NPC（终端、编辑器）都认识你。

安装完成后，打开终端（Mac 上叫「终端」，Windows 上叫「命令提示符」或 PowerShell），输入：

```
python --version
```

如果出现类似 `Python 3.12.0` 的字样，铁匠铺搭好了。

### 运行你的第一行 Python

在终端里输入 `python`（或 `python3`），回车。你会看到三个大于号：

```
>>>
```

这是 Python 的「交互模式」——你敲一行，它立刻回你一行。像是跟魔法书对话：你说一句咒语，它立刻反馈效果。

试试最简单的计算：

```python
1 + 1
```

回车，魔法书回你：

```
2
```

现在，敲出勇者的第一句宣言：

```python
print("Hello World！")
```

回车——屏幕上出现：

```
Hello World！
```

**逐行解读**：

- `print()` —— Python 内置的「喊话」指令，把括号里的东西显示到屏幕上。
- `"Hello World！"` —— 一串文字（字符串），用双引号包住。这是你要喊的内容。
- 整句翻译成人话：「Python，帮我把『Hello World！』这两个字打印到屏幕上。」

**这是冒险的第一个信号弹。就一行字，但它意味着代码世界承认了你的存在。**

### 故意写错一次——报错不是死咒

试试故意少写一个引号：

```python
print("Hello World！)
```

回车后，红色字出现了：

```
SyntaxError: unterminated string literal
```

别慌。这不是「你失败了」，这是魔法书在说：

> 「咒语没念完，我听不懂——你的引号只开没关。补上右边的引号再念一次。」

**逐行解读**：

- `SyntaxError` —— 语法错误。你的咒语不符合 Python 的「语法规则」。
- `unterminated string literal` —— 字符串没结束。你开了引号但忘了关。

以后每次看到红色报错，第一反应不是「我写错了」，而是「魔法书在告诉我哪里不对」。你甚至可以让 AI 帮你解读报错：

> **Prompt 示例**：「我运行 Python 代码出现这个错误：`SyntaxError: unterminated string literal`，代码是 `print("Hello World！)`。这是什么意思？怎么修？」

### 注释——写给冒险者看的笔记

注释不是给计算机看的，是给人看的。计算机执行代码时会直接跳过注释——就像怪物不会偷看你的冒险笔记。

```python
# 勇者初始属性——出门别忘了分配属性点！
hp = 100    # 生命值
mp = 50     # 魔法值
atk = 15    # 物理攻击力

# 这个计算含了装备加成，不要再手动加一次
final_atk = atk + weapon_bonus + buff_bonus
```

**逐行解读**：

- `# 勇者初始属性——出门别忘了分配属性点！` —— 以 `#` 开头，整行都是注释。Python 直接跳过。
- `hp = 100    # 生命值` —— 行内注释。`#` 前面的代码正常执行，`#` 后面的文字被跳过。
- `final_atk = atk + weapon_bonus + buff_bonus` —— 这行目前是伪代码示意（`weapon_bonus` 和 `buff_bonus` 还没定义），注释说明了「这个计算含装备加成，不要重复加」——防止后面写代码的人（包括未来的你）搞错。

**冒险笔记**：你在背包里记的「宝箱在枯树第三根树枝下」——不是给怪物看的，是给未来的自己看的。注释是写给未来自己、队友、AI 伙伴的。

注释的三种用法：

1. **解释为什么这么做**（不是做了什么）——代码本身说了「做什么」，注释说「为什么」。
2. **标记待办和注意事项**——`# TODO: 后面要加暴击判定`。
3. **给 AI 提供上下文**——AI 读注释后能更准确地理解你的意图。

## AI 协作指南

### 场景一：装 Python 遇到问题

装环境是每个编程新手的第一道坎。别自己死磕——让 AI 帮你。

> **Prompt 示例**：「我在 Windows 上安装 Python 3.12，安装完成后在终端输入 `python` 显示『不是内部或外部命令』，怎么做？」

AI 会告诉你：安装时没勾「Add Python to PATH」，然后给你两种解决方法（重装勾上、或者手动添加环境变量）。

### 场景二：让 AI 解读报错

以后你遇到的每一个报错，都可以直接丢给 AI：

> **Prompt 示例**：「我运行 Python 代码时遇到这个错误，帮我解释是什么意思、怎么改：`TypeError: can only concatenate str (not "int") to str`。我的代码是 `print("伤害：" + 25)`。」

AI 会告诉你「文字和数字不能直接加，要用 `str(25)` 转成文字」——顺便帮你记住这个坑，以后不再踩。

## 实战练习

### 练习1：改一行代码，看输出变化（最简单）

打开终端，进入 Python 交互模式（`python`），逐行输入：

```python
print("勇者刘老三")
print("冒险开始！")
print(100)
```

每行输出什么？现在试试：

```python
print("勇者刘老三", "冒险开始！")
print("HP:", 100)
```

`print()` 里用逗号隔开多个东西，输出有什么不同？把你的发现写进注释里。

### 练习2：加一个新功能——写一段带注释的角色初始化（中等）

创建一个 Python 文件（`.py`），命名为 `hero_init.py`。写入以下内容：

```python
# === 勇者刘老三·角色初始化 ===
# 创建时间：今天

# --- 基础属性 ---
hp = 100    # 生命值
mp = 50     # 魔法值
atk = 15    # 物理攻击力
def_stat = 10   # 防御力

# --- 打印角色卡 ---
print("=== 勇者刘老三·角色卡 ===")
print("HP:", hp)
print("MP:", mp)
print("ATK:", atk)
```

用终端运行这个文件：`python hero_init.py`。观察输出。

然后在注释下方加两行新的属性（比如 `gold = 500` 金币和 `level = 1` 等级），并在 `print` 区域补上对应的打印语句。运行看看效果。

### 练习3：结合 AI，让 AI 帮你扩展（综合）

把你练习 2 写的 `hero_init.py` 内容发给 AI：

> **Prompt 示例**：「这是我的 Python 角色初始化代码：{粘贴你的代码}。请帮我加一个『受伤』功能：勇者受到 20 点伤害，HP 减少 20，但防御力可以减少伤害（受到的伤害 = 20 - 防御力，最少受到 1 点伤害）。加注释说明。另外帮我检查我写的注释有没有不准确的地方。」

AI 会给你扩展后的代码。把 AI 的代码保存为新文件 `hero_init_v2.py`，运行看效果。试着理解 AI 每一行在干什么——不懂的直接追问 AI：「`max(1, damage)` 是什么意思？」
